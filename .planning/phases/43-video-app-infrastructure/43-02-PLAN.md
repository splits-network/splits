  <name>Task 1: Token exchange client, types, and hook</name>
  <files>
    apps/video/src/lib/call-api.ts

  <action>
    **types.ts**: Define the video app's local types:
    - `ExchangeResult`: The response from token exchange. Contains `livekit_token: string` and `call` object with: `id`, `title`, `call_type`, `status`, `scheduled_at`, `livekit_room_name`, `participants` array (each with `id`, `user_id`, `role`, and nested `user` object with `first_name`, `last_name`, `avatar_url`, `email`), and `entity_links` array (each with `entity_type`, `entity_id`). Match the shape returned by call-service's `TokenService.exchangeToken()` which returns `{ livekit_token, call }` where `call` is the Call type.
    - `JoinState`: Union type for join flow states: `'loading' | 'confirming' | 'joining' | 'error'`
    - `JoinError`: `{ type: 'invalid' | 'expired' | 'not-started' | 'unknown'; message: string }`

    **call-api.ts**: Export `exchangeToken(token: string): Promise<ExchangeResult>`. Uses `NEXT_PUBLIC_API_URL` env var (default `http://localhost:3000`). Makes POST to `/api/v2/calls/exchange-token` with `{ token }` body. Returns `data` from `{ data: ExchangeResult }` response envelope. On error: parse response JSON for error message. Map HTTP status codes: 404 -> 'invalid', 410 -> 'expired', 400 -> 'not-started', else -> 'unknown'. Throw an object matching `JoinError` shape.

    **use-call-token.ts**: Hook `useCallToken(token: string)`. Returns `{ state: JoinState, result: ExchangeResult | null, error: JoinError | null }`. On mount, calls `exchangeToken(token)`. On success, sets state to `'confirming'` and populates result. On error, sets state to `'error'` and populates error. No retry logic -- user must get a new link if token fails.
  </action>

  </done>

    **splash-screen.tsx**: Client component. Shows branded splash screen while token exchange is loading. Displays brand logo (from `useBrand()`) centered on screen with a subtle CSS loading animation (use a simple pulse or fade animation via CSS keyframes -- no GSAP). Background uses `bg-base-200`. Logo should be reasonably large. Below logo, show a small DaisyUI `loading loading-spinner` indicator.

    **error-page.tsx**: Client component. Branded error display. Accepts `error: JoinError` prop. Shows brand logo, an error icon (use FA duotone: `fa-duotone fa-regular fa-circle-exclamation`), a heading based on error type ('Link Invalid', 'Link Expired', 'Call Not Ready', 'Something Went Wrong'), a description message, and a "Return to Portal" link using brand's `portalUrl`. Full-screen centered layout. Use DaisyUI `alert alert-error` styling or similar.

    **identity-confirm.tsx**: Client component. Shows "Joining as [Name]" screen. Accepts `result: ExchangeResult` and `onConfirm: () => void` props. Displays:
    - Brand logo via branded-header
    - Participant name derived from the current user's participant entry (the user whose token was exchanged -- use the first participant or derive from the exchange result)
    - Call title if set, call type badge
    - List of expected participants with names and roles
    - Scheduled time if set
    - A "Join Call" button (DaisyUI `btn btn-primary btn-lg`) that calls `onConfirm`
    - Full-screen centered layout, clean and minimal
    - Responsive: works on mobile with `w-full sm:w-auto` button pattern

    **join-flow.tsx**: Client component. Orchestrates the join flow. Accepts `token: string` prop. Uses `useCallToken(token)` hook. Based on state:
    - `'loading'`: Render `<SplashScreen />`
    - `'error'`: Render `<ErrorPage error={error} />`
    - `'confirming'`: Render `<IdentityConfirm result={result} onConfirm={handleJoin} />`
    - `'joining'`: Navigate to `/call/${result.call.id}` using Next.js `useRouter().push()`, passing the LiveKit token and call data via URL search params or a client-side state mechanism (prefer `sessionStorage` to avoid exposing the JWT in the URL -- store `{ livekitToken, call }` in sessionStorage keyed by callId, then navigate to `/call/${callId}`)

    **join/[token]/page.tsx**: Server component. Validates token param exists and has minimum length (32+ chars). If invalid, redirect to `/error?reason=invalid`. Otherwise render `<JoinFlow token={token} />`. Use Next.js 16 params pattern: `params: Promise<{ token: string }>`.

    **error/page.tsx**: Server component. Reads `reason` from search params. Renders `<ErrorPage>` with appropriate error based on reason query param. Default to 'unknown' if no reason provided. Use Next.js 16 searchParams pattern: `searchParams: Promise<{ reason?: string }>`.
