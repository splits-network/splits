**call-type-selector.tsx** (~60 lines):
Create a component `CallTypeSelector` with props:
- `value: string` — current call type slug
- `onChange: (slug: string) => void`
- `entityLinks?: Array<{ entity_type: string }>` — for context inference

The component renders a DaisyUI `select` dropdown with a `fieldset` wrapper (per form-controls guidance). Options:
- `interview` — "Interview"
- `recruiting_call` — "Recruiting Call"
- `client_meeting` — "Client Meeting"
- `internal_call` — "Internal Call"

Export a helper function `inferCallType(entityLinks)` that returns the default:
- If any entity link has `entity_type === 'application'`, return `'interview'`
- If any entity link has `entity_type === 'company'`, return `'client_meeting'`
- Otherwise return `'recruiting_call'`

**recording-controls.tsx** (~120 lines):
Create a component `RecordingControls` with props:
- `recordingEnabled: boolean`
- `onRecordingChange: (enabled: boolean) => void`
- `transcriptionEnabled: boolean`
- `onTranscriptionChange: (enabled: boolean) => void`
- `aiAnalysisEnabled: boolean`
- `onAiAnalysisChange: (enabled: boolean) => void`
- `planTier: 'starter' | 'pro' | 'partner'`

Layout:
1. A DaisyUI `toggle` for "Record this call" with label. Default off.
   - When toggled on, show an inline hint below: "Participants will be asked for consent before joining" (text-sm text-base-content/50, with a small info icon).

2. Tier-dependent rendering below the recording toggle:

   - When `planTier` is `'starter'` (Free tier):
     - Recording toggle works normally (Free users CAN record).
     - Instead of separate transcription and AI toggles, show a **single combined row**: "Transcription & AI Analysis" label with a clickable badge: `<a href="/portal/profile?section=subscription" className="badge badge-primary badge-sm ml-2">Upgrade to Pro</a>`. No toggle controls — just the label + badge as a single upgrade prompt row.
     - Below, show a subtle note: "Free plan: recordings available for 7 days. Upgrade for transcription and AI analysis."

   - When `planTier` is `'pro'`:
     - Recording toggle works normally
     - Transcription toggle visible, disabled when recording is off. Default off. Label: "Transcribe this call"
     - AI Analysis toggle visible but disabled with badge: `<a href="/portal/profile?section=subscription" className="badge badge-primary badge-sm ml-2">Upgrade to Partner</a>`

   - When `planTier` is `'partner'`:
     - Recording toggle works normally
     - Transcription toggle visible, disabled when recording is off. Default off. Label: "Transcribe this call"
     - AI Analysis toggle visible, disabled when recording or transcription is off. Default off.
     - No badges, all toggles functional

Use DaisyUI classes throughout (toggle, badge). No custom styling for toggle behavior — use the native DaisyUI `toggle` input with `toggle-primary` class.

Wrap the whole section in a `fieldset` with legend "Recording & AI".
