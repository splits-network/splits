**ParticipantSidebar (new file):**
Create an always-visible sidebar component showing all participants:
- Props: `participants: RemoteParticipant[]`, `interviewContext: InterviewContext`, `localName: string`
- Renders a vertical list on the right side of the screen
- Each entry shows: avatar (small circle, use CameraOffFallback pattern for initials), name, role badge ("Candidate" in accent color, "Interviewer" in neutral), mute indicator (mic icon with slash when muted)

- Props: `trackRef: TrackReference`

**VideoRoom refactor:**
The current VideoRoom only handles 1 remote participant (`remoteParticipants[0]`). Refactor to:

1. Use `useRemoteParticipants()` to get ALL remote participants
2. Use `useTracks([{ source: Track.Source.ScreenShare, withPlaceholder: false }])` to detect screen share tracks
3. Layout logic:
   - **No screen shares**: Grid layout for all participants. Use CSS grid:
     - 1 participant: single full-area tile
     - 2 participants: `grid-cols-2` (side by side)
     - 3-4 participants: `grid-cols-2 grid-rows-2`
     - 5-6 participants: `grid-cols-3 grid-rows-2`
     - 7+ participants: `grid-cols-3` with overflow scroll
   - **Screen share active**: Screen share takes main area (~75% width), participant videos shrink to a vertical strip on the left (`w-48`, small tiles stacked vertically)
   - **Multiple screen shares**: Screen shares tile in the main area as a grid, participant strip remains
4. Keep SelfViewPip overlay for local user
5. Add ParticipantSidebar on the right edge (always visible)
6. Keep existing props interface but make it work with 0-N remote participants
7. When a participant joins/leaves, show a toast notification using a simple div with CSS animation (fade in/out). Track previous participant count via useRef to detect changes.

**VideoControls update:**
Add a screen share toggle button between the camera toggle and the device settings dropdown:

```tsx
{/* Screen share toggle */}
<TrackToggle
    source={Track.Source.ScreenShare}
    className="btn btn-circle btn-sm lg:btn-md data-[lk-enabled=true]:btn-success"
    showIcon={false}
>
    <i className="fa-duotone fa-regular fa-screen-share" />
</TrackToggle>
```

LiveKit's `TrackToggle` with `Track.Source.ScreenShare` handles the browser's screen share picker dialog automatically. When sharing, the button gets the `data-[lk-enabled=true]` state which we style with `btn-success` to indicate active sharing.

If `TrackToggle` doesn't support ScreenShare source natively, use `useLocalParticipant()` and manually toggle via:
```tsx
const { localParticipant } = useLocalParticipant();
const handleScreenShare = async () => {
    const enabled = localParticipant.isScreenShareEnabled;
    await localParticipant.setScreenShareEnabled(!enabled);
};
```
Wrap in a button with state tracking.
