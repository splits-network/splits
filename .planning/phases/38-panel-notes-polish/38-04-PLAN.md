  <action>
Create a new tab component that displays all interviews for an application:

**Data fetching:**
- On mount, fetch `GET /api/v2/interviews?application_id={applicationId}&include_context=true` using the authenticated client pattern (useAuth + createAuthenticatedClient)
- Handle loading state with `loading loading-spinner`
- Handle empty state: "No interviews scheduled" with a camera icon

**Interview card layout (use a card-based timeline design):**
Each interview renders as a card in a vertical timeline:

```
<div className="space-y-4">
  {interviews.map((interview, index) => (
    <InterviewCard key={interview.id} interview={interview} roundNumber={index + 1} />
  ))}
</div>
```

**InterviewCard contents:**
- **Header row**: Round indicator ("Round {n}" or round_name if set, e.g., "Round 2: Technical Screen") + status badge
  - Status badge colors: scheduled=info, in_progress=warning, completed=success, cancelled=error, no_show=neutral, expired=neutral
  - Use `badge badge-{color} badge-sm`
- **Date/time**: Formatted scheduled_at (e.g., "Mar 8, 2026 at 2:00 PM") + duration ("60 min")
- **Participants**: Small avatar circles in a row with names on hover (tooltip). Candidate gets accent ring.
  - Use `avatar-group` DaisyUI pattern or simple flex row of `w-8 h-8` circles
  - Show role on hover/tooltip
- **Assets row** (only for completed interviews): Icons/links for available assets
  - Recording: `fa-circle-play` icon, links to playback (if recording_blob_url exists)
  - Transcript: `fa-file-lines` icon, clickable to open transcript panel
  - AI Summary: `fa-brain` icon with "AI Summary" text, clickable to expand inline or link to notes tab
  - Each icon is muted if not available, colored if available
- **Interview type badge**: Small text showing interview_type (e.g., "Panel", "Technical")

**Card styling:**
- `bg-base-100 border border-base-300 p-4` (no rounded corners per Basel rules)
- `border-l-4 border-primary` on the most recent/upcoming interview
- `opacity-60` on cancelled interviews

**Timeline connector:**
- Vertical line connecting cards: `border-l-2 border-base-300 ml-4` between cards

**Recording playback link:**
For interviews with recordings, the play icon should open the recording in a new tab or trigger the existing playback component. Use `window.open` to the interview page or construct the playback URL. Check how Phase 36 built playback -- it's likely on the interview detail route or a modal.

**Transcript/Summary display:**
- Transcript: Link to the existing transcript panel (built in Phase 37). Could open a collapse/accordion inline showing the transcript text, or link to the notes tab filtered by interview_summary type.
- AI Summary: If has_summary is true, show a collapsible section with the summary preview. Fetch the full summary on expand. OR simply link to the Notes tab where the interview_summary note lives.
- Simplest approach: For both transcript and summary, show status indicators and link to the relevant tabs. Use DaisyUI `collapse` for inline expansion if showing content directly.

1. Add state: `const [roundName, setRoundName] = useState("");`
2. In the confirm step (step === "confirm"), add a text input before or after the existing fields:
   ```tsx
   <fieldset>
       <legend className="fieldset-legend text-sm">Round Name (optional)</legend>
       <input
           type="text"
           className="input input-sm w-full"
           placeholder="e.g., Technical Screen, Culture Fit"
           value={roundName}
           onChange={(e) => setRoundName(e.target.value)}
           maxLength={100}
       />
       <p className="text-sm text-base-content/50 mt-1">
           Optional label for this interview round
       </p>
   </fieldset>
   ```

Note: The existing `title` field on interviews may already serve a similar purpose. Check if `title` is used in the schedule modal. If so, `round_name` is additive -- title is the calendar event title, round_name is the round label. They serve different purposes.
