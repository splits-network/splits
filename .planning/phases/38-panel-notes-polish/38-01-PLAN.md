Create a migration that:

1. Adds `round_name TEXT` column to the `interviews` table (nullable, optional label like "Technical Screen", "Culture Fit"). No default.

2. Creates `interview_notes` table:
```sql
CREATE TABLE interview_notes (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    interview_id UUID NOT NULL REFERENCES interviews(id) ON DELETE CASCADE,
    participant_id UUID NOT NULL REFERENCES interview_participants(id) ON DELETE CASCADE,
    user_id UUID NOT NULL REFERENCES users(id),
    content TEXT NOT NULL DEFAULT '',
    updated_at TIMESTAMPTZ NOT NULL DEFAULT now(),
    created_at TIMESTAMPTZ NOT NULL DEFAULT now()
);
```
- Unique constraint on (interview_id, participant_id) -- one note document per participant per interview
- Index on interview_id for fast lookup
- updated_at trigger using existing `update_updated_at_column()` function

This is an upsert-friendly design: each participant has one note row that gets updated repeatedly (auto-save), not multiple note rows.
