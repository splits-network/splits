**useInterviewNotes hook:**
Create a hook that manages in-call note state with debounced auto-save:
- Parameters: `interviewId: string`, `apiBase: string`, `getToken: () => Promise<string | null>`, `options?: { magicLinkToken?: string }`
- State: `content` (string), `saving` (boolean), `lastSaved` (Date | null), `error` (string | null)
- On mount: `GET /api/v2/interviews/{id}/notes` to load existing note content for this participant (if any). For magic link users, pass `?token={magicLinkToken}` as auth.
- On content change: debounce 2 seconds, then `PUT /api/v2/interviews/{id}/notes` with `{ content }`. For authenticated users, use Bearer token from `getToken()`. For magic link users, include token in body.
- Expose: `{ content, setContent, saving, lastSaved, error, postToApplication }`
- `postToApplication()`: calls `POST /api/v2/interviews/{id}/notes/post-to-application` -- only callable by interviewers. This is called on disconnect.
- Use `useRef` for the debounce timer, clean up on unmount
- Flush any pending save on unmount (call save immediately if dirty)

**NotesPanel component:**
- Props: `isOpen: boolean`, `onToggle: () => void`, `interviewId: string`, `apiBase: string`, `getToken: () => Promise<string | null>`, `magicLinkToken?: string`
- When `isOpen` is false, render nothing (the toggle button is on the controls bar)
- When `isOpen` is true, render a right-side panel:
  - Width: `w-96` on desktop, `w-full` on mobile (overlay)
  - Header: "Interview Notes" title + close button + save status indicator ("Saving...", "Saved at {time}", "Unsaved changes")
  - Body: Use `MarkdownEditor` from `@splits-network/shared-ui` with:
    - `height={400}` (or flex to fill available space)
    - `preview="edit"` (write mode by default during call)
    - `placeholder="Take notes during the interview..."`
    - No label, no helper text (compact mode)
  - The MarkdownEditor is imported dynamically or from shared-ui. Since shared-video shouldn't depend on shared-ui directly, use a render prop or slot pattern:

    Actually, reconsider: shared-video already depends on @livekit packages. Adding shared-ui as a peer dep is fine since both portal and candidate apps have it. Add `@splits-network/shared-ui` as a peerDependency in shared-video's package.json.

    OR simpler: Use a plain `<textarea>` with markdown support indication instead of the full MDEditor. The CONTEXT.md says "Reuse the existing markdown editor component" but in a video call context, a simpler textarea may be more practical.

    Decision: Use a plain textarea with monospace font for notes. The markdown editor has too much chrome for an in-call panel. The notes content will be rendered as markdown when posted to application notes. Style the textarea: `textarea textarea-ghost w-full h-full font-mono text-sm resize-none focus:outline-none`.

- Styling: `bg-base-100 border-l border-base-300 shadow-lg` panel
- Transition: slide in from right with CSS transition (`transform transition-transform duration-200`)

**Portal InterviewClient (interview-client.tsx):**
1. Add state: `const [notesOpen, setNotesOpen] = useState(false);`
2. Import `NotesPanel` and `useInterviewNotes` from shared-video
3. Initialize the notes hook: `const notes = useInterviewNotes(interviewId, API_BASE, getToken);`
4. Pass notes toggle props to VideoRoom/VideoControls:
   - `onNotesToggle={() => setNotesOpen(!notesOpen)}`
   - `notesOpen={notesOpen}`
5. Pass NotesPanel as render prop to VideoRoom:
   ```tsx
   notesPanel={notesOpen ? (
       <NotesPanel
           isOpen={notesOpen}
           onToggle={() => setNotesOpen(false)}

           apiBase={API_BASE}
           getToken={getToken}
       />
   ```
