**service.ts** — Create `CallService` class following InterviewService pattern:

Constructor takes `repository: CallRepository, eventPublisher: IEventPublisher`.

Methods:

1. `createCall(input: CreateCallInput, createdByClerkUserId: string)` -> `CallWithParticipants`
   - Resolve Clerk user ID to internal user ID via `repository.resolveUserId()`
   - Validate call_type exists (query call_types table or trust FK constraint — prefer trusting DB)
   - If scheduled_at provided, validate it's in the future
   - Insert call record with `created_by` = resolved user ID
   - Bulk insert entity_links from `input.entity_links` array
   - Bulk insert participants from `input.participants` array. Auto-add the creator as `host` if not in the participants list
   - Publish `call.created` event via eventPublisher with `{ call_id, call_type, created_by, entity_links, participant_count }`
   - Return the created call with participants and entity links

2. `listCalls(params: StandardListParams, filters: CallListFilters)` -> `StandardListResponse<CallWithParticipants>`
   - Delegate to repository.listCalls()

3. `getCallDetail(id: string, include?: string[])` -> `CallDetail`
   - Delegate to repository.getCallDetail()
   - Throw 404 if not found

4. `updateCall(id: string, input: UpdateCallInput)` -> `Call`
   - Validate call exists and is not deleted
   - If updating scheduled_at, validate it's in the future and call status is 'scheduled'
   - Delegate to repository.updateCall()

5. `deleteCall(id: string)` -> `void`
   - Validate call exists
   - Delegate to repository.deleteCall() (soft delete)

6. `startCall(id: string)` -> `Call`
   - Validate call exists with status 'scheduled'
   - Generate a unique livekit_room_name: `call-${id}-${Date.now()}`
   - Update status to 'active', set started_at = now(), set livekit_room_name
   - Publish `call.started` event with `{ call_id, call_type, room_name, started_at }`

7. `endCall(id: string)` -> `Call`
   - Validate call exists with status 'active'
   - Calculate duration_minutes from started_at to now
   - Update status to 'completed', set ended_at = now(), set duration_minutes
   - Publish `call.ended` event with `{ call_id, call_type, duration_minutes, ended_at }`

8. `cancelCall(id: string)` -> `Call`
   - Validate call status is 'scheduled' or 'active'
   - Update status to 'cancelled'
   - Publish `call.cancelled` event with `{ call_id, call_type, cancelled_at }`

9. `addParticipant(callId: string, input: { user_id: string, role: CallParticipantRole })` -> `CallParticipant`
   - Validate call exists and is not deleted
   - Delegate to repository

10. `removeParticipant(callId: string, participantId: string)` -> `void`
    - Validate participant belongs to this call
    - Delegate to repository

11. `addEntityLink(callId: string, input: { entity_type: CallEntityType, entity_id: string })` -> `CallEntityLink`
    - Validate call exists and is not deleted
    - Delegate to repository

12. `removeEntityLink(callId: string, linkId: string)` -> `void`
    - Validate link belongs to this call
    - Delegate to repository

All error throwing uses `Object.assign(new Error('message'), { statusCode: NNN })` pattern matching video-service.

**token-service.ts** — Create `TokenService` class:

Constructor takes `repository: CallRepository, livekitApiKey: string, livekitApiSecret: string`.

Methods:

1. `createToken(callId: string, userClerkId: string)` -> `{ token: string, livekit_token: string }`
   - Resolve Clerk user ID to internal user ID
   - Validate user is a participant of this call
   - Get the call record (need livekit_room_name and call_type)
   - Generate a random access token (crypto.randomBytes(32).toString('hex'))
   - Store in call_access_tokens with 24h expiry
   - Generate a LiveKit JWT using `livekit-server-sdk` AccessToken class:
     - Identity = user_id
     - Room = call's livekit_room_name (if null, throw 400 "Call has not started yet")
     - Grants: roomJoin = true, canPublish = true, canSubscribe = true
     - TTL = 4 hours
   - Return both the magic link token and the LiveKit JWT

2. `exchangeToken(token: string)` -> `{ livekit_token: string, call: Call }`
   - Look up access token by token string
   - Validate not expired (expires_at > now)
   - Mark as used (used_at = now)
   - Get the call record
   - Generate LiveKit JWT for the token's user_id
   - Return the LiveKit token and call details

Follow the video-service TokenService pattern for LiveKit JWT generation.

  <action>
Replace the stub routes.ts with full route registration. Follow the video-service registerInterviewRoutes pattern.

**Route function:** `registerV2Routes(app: FastifyInstance, config: RegisterConfig)`

Config interface includes: supabaseUrl, supabaseKey, rabbitMqUrl, eventPublisher (IEventPublisher), livekitApiKey, livekitApiSecret, livekitWsUrl.

Inside: instantiate `CallRepository`, `CallService`, `TokenService`, then register all routes.

**Extract Clerk user ID from request:** Use `request.headers['x-clerk-user-id']` (the gateway forwards this). If missing, return 401.

**Routes (all under /api/v2/calls prefix):**

1. `GET /api/v2/calls` — List calls
   - Parse query params: page, limit, sort_by, sort_order (StandardListParams) + call_type, entity_type, entity_id, status, date_from, date_to (CallListFilters)
   - Call `service.listCalls(params, filters)`
   - Return `{ data: result.data, pagination: result.pagination }`

2. `GET /api/v2/calls/:id` — Get call detail
   - Parse query param `include` (comma-separated string: recordings,transcript,summary,notes)
   - Call `service.getCallDetail(id, includeArray)`
   - Return `{ data: call }`

3. `POST /api/v2/calls` — Create call
   - Parse body: `{ call_type, title?, scheduled_at?, entity_links: [{entity_type, entity_id}], participants: [{user_id, role}] }`
   - Call `service.createCall(input, clerkUserId)`
   - Return 201 `{ data: call }`

4. `PUT /api/v2/calls/:id` — Update call
   - Parse body: `{ title?, scheduled_at? }`
   - Call `service.updateCall(id, input)`
   - Return `{ data: call }`

5. `DELETE /api/v2/calls/:id` — Soft delete call
   - Call `service.deleteCall(id)`
   - Return 204

6. `POST /api/v2/calls/:id/start` — Start call
   - Call `service.startCall(id)`
   - Return `{ data: call }`

7. `POST /api/v2/calls/:id/end` — End call
   - Call `service.endCall(id)`
   - Return `{ data: call }`

8. `POST /api/v2/calls/:id/cancel` — Cancel call
   - Call `service.cancelCall(id)`
   - Return `{ data: call }`

9. `POST /api/v2/calls/:id/participants` — Add participant
   - Parse body: `{ user_id, role }`
   - Call `service.addParticipant(id, input)`
   - Return 201 `{ data: participant }`

10. `DELETE /api/v2/calls/:id/participants/:participantId` — Remove participant
    - Call `service.removeParticipant(id, participantId)`
    - Return 204

11. `POST /api/v2/calls/:id/entities` — Add entity link
    - Parse body: `{ entity_type, entity_id }`
    - Call `service.addEntityLink(id, input)`
    - Return 201 `{ data: link }`

12. `DELETE /api/v2/calls/:id/entities/:linkId` — Remove entity link
    - Call `service.removeEntityLink(id, linkId)`
    - Return 204

13. `POST /api/v2/calls/:id/token` — Generate access token + LiveKit JWT
    - Call `tokenService.createToken(id, clerkUserId)`
    - Return `{ data: { token, livekit_token } }`

All routes follow the `{ data }` response envelope per api-response-format.md. All routes use try/catch with error.statusCode propagation matching video-service pattern.

**Wire into index.ts:** Update the `registerV2Routes` call in index.ts to pass all required config (supabaseUrl, supabaseKey, eventPublisher, livekitApiKey, livekitApiSecret, livekitWsUrl). The config interface should match what routes.ts expects.
