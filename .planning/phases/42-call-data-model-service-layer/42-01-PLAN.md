  <name>Task 1: Create call system migration</name>
  <files>supabase/migrations/20260312000001_create_call_tables.sql</files>
  <action>
Create a single migration file with all call-related schema objects. Follow the exact pattern from `20260307000005_create_interviews_schema.sql` (section comments, enum creation, table creation, indexes, triggers).

**Enums to create:**
- `call_status` — `scheduled`, `active`, `completed`, `cancelled`
- `call_entity_type` — `application`, `job`, `company`, `firm`, `candidate`
- `call_participant_role` — `host`, `participant`, `observer`

**Tables to create (in order):**

1. `call_types` (lookup table, NOT an enum):
   - `slug TEXT PRIMARY KEY` (e.g., 'interview', 'client_meeting', 'internal_call')
   - `label TEXT NOT NULL` (human-readable display name)
   - `default_duration_minutes INT NOT NULL DEFAULT 60`
   - `requires_recording_consent BOOLEAN NOT NULL DEFAULT true`
   - `supports_ai_summary BOOLEAN NOT NULL DEFAULT true`
   - `created_at TIMESTAMPTZ NOT NULL DEFAULT now()`
   - Seed with three rows: `('interview', 'Interview', 60, true, true)`, `('client_meeting', 'Client Meeting', 30, true, true)`, `('internal_call', 'Internal Call', 30, false, false)`

2. `calls`:
   - `id UUID PRIMARY KEY DEFAULT gen_random_uuid()`
   - `call_type TEXT NOT NULL REFERENCES call_types(slug)`
   - `status call_status NOT NULL DEFAULT 'scheduled'`
   - `title TEXT` (optional subject/description)
   - `livekit_room_name TEXT UNIQUE` (nullable — assigned when call starts)
   - `scheduled_at TIMESTAMPTZ`
   - `started_at TIMESTAMPTZ`
   - `ended_at TIMESTAMPTZ`
   - `duration_minutes INT`
   - `created_by UUID NOT NULL REFERENCES users(id)`
   - `deleted_at TIMESTAMPTZ` (soft delete)
   - `created_at TIMESTAMPTZ NOT NULL DEFAULT now()`
   - `updated_at TIMESTAMPTZ NOT NULL DEFAULT now()`
   - Indexes: `idx_calls_status` partial on `(status) WHERE status IN ('scheduled', 'active')`, `idx_calls_call_type` on `(call_type)`, `idx_calls_scheduled_at` on `(scheduled_at) WHERE status = 'scheduled'`, `idx_calls_created_by` on `(created_by)`
   - updated_at trigger: reuse `update_updated_at_column()` function (already exists from interviews migration)

3. `call_entity_links`:
   - `id UUID PRIMARY KEY DEFAULT gen_random_uuid()`
   - `call_id UUID NOT NULL REFERENCES calls(id) ON DELETE CASCADE`
   - `entity_type call_entity_type NOT NULL`
   - `entity_id UUID NOT NULL`
   - `created_at TIMESTAMPTZ NOT NULL DEFAULT now()`
   - `CONSTRAINT uq_call_entity_link UNIQUE (call_id, entity_type, entity_id)`
   - Indexes: `idx_call_entity_links_entity` on `(entity_type, entity_id)` — this is the critical polymorphic query index
   - Index: `idx_call_entity_links_call` on `(call_id)`

4. `call_participants`:
   - `id UUID PRIMARY KEY DEFAULT gen_random_uuid()`
   - `call_id UUID NOT NULL REFERENCES calls(id) ON DELETE CASCADE`
   - `user_id UUID NOT NULL REFERENCES users(id)`
   - `role call_participant_role NOT NULL`
   - `joined_at TIMESTAMPTZ`
   - `left_at TIMESTAMPTZ`
   - `consent_given BOOLEAN NOT NULL DEFAULT false`
   - `consent_at TIMESTAMPTZ`
   - `created_at TIMESTAMPTZ NOT NULL DEFAULT now()`
   - `CONSTRAINT uq_call_participant UNIQUE (call_id, user_id)`
   - Indexes: `idx_call_participants_user` on `(user_id)`, `idx_call_participants_call` on `(call_id)`

5. `call_access_tokens`:
   - `id UUID PRIMARY KEY DEFAULT gen_random_uuid()`
   - `call_id UUID NOT NULL REFERENCES calls(id) ON DELETE CASCADE`
   - `user_id UUID NOT NULL REFERENCES users(id)`
   - `token TEXT NOT NULL UNIQUE`
   - `expires_at TIMESTAMPTZ NOT NULL`
   - `used_at TIMESTAMPTZ`
   - `created_at TIMESTAMPTZ NOT NULL DEFAULT now()`
   - Index: `idx_call_access_tokens_token` on `(token)`
   - Index: `idx_call_access_tokens_call` on `(call_id)`

6. `call_recordings` (support multiple recordings per call — a call may be stopped/restarted):
   - `id UUID PRIMARY KEY DEFAULT gen_random_uuid()`
   - `call_id UUID NOT NULL REFERENCES calls(id) ON DELETE CASCADE`
   - `recording_status TEXT NOT NULL DEFAULT 'pending'` CHECK constraint: `pending`, `recording`, `processing`, `ready`, `failed`
   - `egress_id TEXT`
   - `blob_url TEXT`
   - `duration_seconds INT`
   - `file_size_bytes BIGINT`
   - `started_at TIMESTAMPTZ`
   - `ended_at TIMESTAMPTZ`
   - `created_at TIMESTAMPTZ NOT NULL DEFAULT now()`
   - Index: `idx_call_recordings_call` on `(call_id)`

7. `call_transcripts`:
   - `id UUID PRIMARY KEY DEFAULT gen_random_uuid()`
   - `call_id UUID NOT NULL REFERENCES calls(id) ON DELETE CASCADE`
   - `storage_url TEXT NOT NULL` (Supabase Storage URL to JSON file)
   - `transcript_status TEXT NOT NULL DEFAULT 'pending'` CHECK constraint: `pending`, `processing`, `ready`, `failed`
   - `error TEXT`
   - `language TEXT DEFAULT 'en'`
   - `created_at TIMESTAMPTZ NOT NULL DEFAULT now()`
   - `updated_at TIMESTAMPTZ NOT NULL DEFAULT now()`
   - Index: `idx_call_transcripts_call` on `(call_id)`
   - updated_at trigger

8. `call_summaries`:
   - `id UUID PRIMARY KEY DEFAULT gen_random_uuid()`
   - `call_id UUID NOT NULL REFERENCES calls(id) ON DELETE CASCADE`
   - `summary JSONB NOT NULL` (type-specific schema: interview=strengths/concerns/recommendation, client_meeting=action_items/decisions/follow_ups)
   - `summary_status TEXT NOT NULL DEFAULT 'pending'` CHECK constraint: `pending`, `processing`, `ready`, `failed`
   - `error TEXT`
   - `model TEXT` (AI model used)
   - `created_at TIMESTAMPTZ NOT NULL DEFAULT now()`
   - `updated_at TIMESTAMPTZ NOT NULL DEFAULT now()`
   - `CONSTRAINT uq_call_summary UNIQUE (call_id)` — one summary per call, re-running replaces
   - updated_at trigger

9. `call_notes`:
   - `id UUID PRIMARY KEY DEFAULT gen_random_uuid()`
   - `call_id UUID NOT NULL REFERENCES calls(id) ON DELETE CASCADE`
   - `user_id UUID NOT NULL REFERENCES users(id)`
   - `content TEXT NOT NULL`
   - `created_at TIMESTAMPTZ NOT NULL DEFAULT now()`
   - `updated_at TIMESTAMPTZ NOT NULL DEFAULT now()`
   - `CONSTRAINT uq_call_note_per_user UNIQUE (call_id, user_id)` — one note per participant per call
   - Index: `idx_call_notes_call` on `(call_id)`
   - updated_at trigger

**RLS Policies:**
Enable RLS on all tables. Create policies following this pattern:
- `call_types`: SELECT for all authenticated (anon key can read lookup data)
- `calls`: SELECT for users who are participants OR stakeholders of linked entities. For Phase 42, implement participant-based access only (entity stakeholder access is Claude's discretion — use a simple "user is participant" check). INSERT/UPDATE/DELETE for authenticated users (service role key bypasses RLS anyway).
- `call_entity_links`, `call_participants`, `call_access_tokens`, `call_recordings`, `call_transcripts`, `call_summaries`, `call_notes`: Inherit from parent call — SELECT/INSERT/UPDATE/DELETE allowed if user can see the parent call (subquery on call_participants).

Use `auth.uid()` for RLS policies consistent with existing Supabase patterns. Since the service uses service_role_key (bypasses RLS), these policies are primarily for direct Supabase client access.

  <action>
Create a small, separate migration that adds the nullable `call_id` FK to the existing `interviews` table. This is deliberately a separate migration to isolate the backward-compatibility change from the new schema.

```sql
-- Add nullable call_id FK to interviews for Phase 46 migration
ALTER TABLE interviews
  ADD COLUMN call_id UUID REFERENCES calls(id);

-- Unique constraint: one interview per call (1:1 relationship)
CREATE UNIQUE INDEX idx_interviews_call_id ON interviews(call_id) WHERE call_id IS NOT NULL;
```

This column is NOT populated in Phase 42. Phase 46 will wire existing interviews to call records. The unique partial index ensures 1:1 mapping while allowing the nullable state.

Do NOT modify any other interview columns, constraints, or code. Zero functional changes to interview system.
