  <action>
Create a SQL migration that establishes the interview data model. Follow existing migration patterns (CREATE TYPE for enums, UUID PKs, timestamptz, indexes).

**Enums:**
- `interview_status` AS ENUM: `scheduled`, `in_progress`, `completed`, `cancelled`, `no_show`, `expired`
- `interview_type` AS ENUM: `screening`, `technical`, `cultural`, `final`, `panel`, `debrief`
- `interview_participant_role` AS ENUM: `interviewer`, `candidate`, `observer`

**Table: `interviews`**
- `id` UUID PK DEFAULT gen_random_uuid()
- `application_id` UUID NOT NULL REFERENCES applications(id) ON DELETE CASCADE
- `room_name` TEXT UNIQUE â€” LiveKit room name, NULL until room created (lazy creation)
- `status` interview_status NOT NULL DEFAULT 'scheduled'
- `interview_type` interview_type NOT NULL DEFAULT 'screening'
- `title` TEXT â€” optional human-readable title
- `scheduled_at` TIMESTAMPTZ NOT NULL â€” when the interview is supposed to start
- `scheduled_duration_minutes` INT NOT NULL DEFAULT 60
- `actual_start_at` TIMESTAMPTZ â€” set when first participant joins (status -> in_progress)
- `actual_end_at` TIMESTAMPTZ â€” set when room closes (status -> completed)
- `cancellation_reason` TEXT â€” optional, set when status = cancelled
- `max_duration_seconds` INT NOT NULL DEFAULT 14400 â€” 4 hours
- `grace_period_seconds` INT NOT NULL DEFAULT 300 â€” 5 min empty room timeout
- `metadata` JSONB DEFAULT '{}' â€” extensible for downstream phases
- `created_by` UUID NOT NULL REFERENCES users(id) â€” who scheduled it
- `created_at` TIMESTAMPTZ NOT NULL DEFAULT now()
- `updated_at` TIMESTAMPTZ NOT NULL DEFAULT now()

Indexes:
- `idx_interviews_application` ON interviews(application_id)
- `idx_interviews_status` ON interviews(status) WHERE status IN ('scheduled', 'in_progress')
- `idx_interviews_scheduled_at` ON interviews(scheduled_at) WHERE status = 'scheduled'

**Table: `interview_participants`**
- `id` UUID PK DEFAULT gen_random_uuid()
- `interview_id` UUID NOT NULL REFERENCES interviews(id) ON DELETE CASCADE
- `user_id` UUID NOT NULL REFERENCES users(id) â€” uses users.id per context decisions
- `role` interview_participant_role NOT NULL
- `joined_at` TIMESTAMPTZ â€” when they actually joined the call
- `left_at` TIMESTAMPTZ â€” when they left
- `created_at` TIMESTAMPTZ NOT NULL DEFAULT now()
- UNIQUE(interview_id, user_id)

Index:
- `idx_interview_participants_user` ON interview_participants(user_id)

**Table: `interview_access_tokens`**
- `id` UUID PK DEFAULT gen_random_uuid()
- `interview_id` UUID NOT NULL REFERENCES interviews(id) ON DELETE CASCADE
- `participant_id` UUID NOT NULL REFERENCES interview_participants(id) ON DELETE CASCADE
- `token` TEXT NOT NULL UNIQUE â€” cryptographically random, URL-safe
- `created_at` TIMESTAMPTZ NOT NULL DEFAULT now()
- UNIQUE(interview_id, participant_id) â€” one token per participant per interview

Index:
- `idx_interview_access_tokens_token` ON interview_access_tokens(token) â€” fast lookup for magic link exchange

**Do NOT add RLS policies** â€” services use service_role_key and bypass RLS.
