  <action>
Build the interview resource following V2 patterns (types -> repository -> service -> routes).

**types.ts:**
```
InterviewStatus = 'scheduled' | 'in_progress' | 'completed' | 'cancelled' | 'no_show' | 'expired'
InterviewType = 'screening' | 'technical' | 'cultural' | 'final' | 'panel' | 'debrief'
ParticipantRole = 'interviewer' | 'candidate' | 'observer'

Interview: { id, application_id, room_name, status, interview_type, title, scheduled_at, scheduled_duration_minutes, actual_start_at, actual_end_at, cancellation_reason, max_duration_seconds, grace_period_seconds, metadata, created_by, created_at, updated_at }

InterviewParticipant: { id, interview_id, user_id, role, joined_at, left_at, created_at }

InterviewAccessToken: { id, interview_id, participant_id, token, created_at }

InterviewWithParticipants: Interview & { participants: InterviewParticipant[] }

CreateInterviewInput: { application_id, interview_type?, title?, scheduled_at, scheduled_duration_minutes?, participants: Array<{ user_id, role }> }
```

**repository.ts (InterviewRepository):**
- Constructor takes supabaseUrl, supabaseKey — creates Supabase client
- `createInterview(input, createdBy)` — insert into interviews, return created row
- `addParticipants(interviewId, participants[])` — bulk insert into interview_participants
- `createAccessToken(interviewId, participantId, token)` — insert into interview_access_tokens
- `findById(id)` — select interview by id
- `findByIdWithParticipants(id)` — select interview + join participants
- `findByApplicationId(applicationId, options?: { status?, limit?, offset? })` — list interviews for an application with pagination
- `findAccessTokenByToken(token)` — lookup token row, joining interview and participant
- `updateStatus(id, status, extraFields?)` — update interview status + optional fields (actual_start_at, actual_end_at, cancellation_reason)
- `updateParticipantJoined(participantId)` — set joined_at = now()
- `updateParticipantLeft(participantId)` — set left_at = now()

Follow Supabase client pattern from chat-service repository (createClient, .from().select(), .insert(), etc.).

**service.ts (InterviewService):**
- Constructor takes repository, eventPublisher
- `createInterview(input, createdByUserId)`:
  1. Validate application_id exists (query applications table)
  2. Validate scheduled_at is in the future
  3. Create interview record via repository
  4. Add participants via repository (bulk insert)
  5. Generate access tokens for each participant (crypto.randomBytes(32).toString('base64url'))
  6. Store tokens via repository
  7. Publish `interview.created` event via eventPublisher
  8. Return interview with participants
- `getInterview(id)`: findByIdWithParticipants, 404 if not found
- `listByApplication(applicationId, options)`: findByApplicationId with pagination
- `cancelInterview(id, reason?, cancelledByUserId?)`:
  1. Find interview, validate current status is 'scheduled' or 'in_progress'
  2. Update status to 'cancelled' with cancellation_reason
  3. Publish `interview.cancelled` event
  4. Return updated interview

**routes.ts:**
- `registerInterviewRoutes(app, deps: { supabaseUrl, supabaseKey, eventPublisher })` function
- Instantiate InterviewRepository and InterviewService
- Routes (all under `/api/v2/interviews`):
  - `POST /api/v2/interviews` — create interview. Body: CreateInterviewInput. Auth: x-clerk-user-id header (set by gateway). Returns 201 + `{ data: InterviewWithParticipants }`
  - `GET /api/v2/interviews/:id` — get interview with participants. Returns `{ data: InterviewWithParticipants }`
  - `GET /api/v2/interviews` — list by application_id (required query param). Returns `{ data: Interview[], pagination: { total, page, limit, total_pages } }`
  - `PATCH /api/v2/interviews/:id/cancel` — cancel interview. Body: `{ reason?: string }`. Returns `{ data: Interview }`

All responses wrapped in `{ data: ... }` envelope per API response format guidance.
All list responses include pagination object.
Get clerk_user_id from `request.headers['x-clerk-user-id']` for auth context.
