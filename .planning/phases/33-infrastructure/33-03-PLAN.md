  <action>
Create a TokenService class and add token-related routes to video-service.

**token-service.ts (TokenService):**
- Constructor takes: repository (InterviewRepository), livekitApiKey (string), livekitApiSecret (string)
- `exchangeMagicLink(token: string)`:
  1. Look up token in interview_access_tokens via repository.findAccessTokenByToken(token)
  2. If not found, throw 404 (invalid/expired link)
  3. Load the interview via repository.findById(tokenRow.interview_id)
  4. If interview status is 'cancelled' or 'expired', throw 410 Gone with friendly message ("This interview has ended or was cancelled")
  5. If interview status is 'completed' or 'no_show', throw 410 Gone
  6. Load the participant via the participant_id on the token
  7. Generate a LiveKit room JWT using `livekit-server-sdk`:
     ```
     import { AccessToken } from 'livekit-server-sdk';
     const at = new AccessToken(apiKey, apiSecret, { identity: participant.user_id, name: participantName });
     at.addGrant({ roomJoin: true, room: roomName, canPublish: true, canSubscribe: true, canPublishData: true });
     // Candidate gets same permissions as interviewer for now (Claude's discretion from context)
     // Token TTL: 4 hours (matches max call duration)
     at.ttl = '4h';
     const jwt = await at.toJwt();
     ```
  8. If interview has no room_name yet (lazy room creation), generate one: `interview-${interview.id}` and update the interview record
  9. Return: `{ jwt, room_name, interview: { id, status, interview_type, scheduled_at }, participant: { id, role } }`

- `generateAuthenticatedToken(interviewId: string, userId: string)`:
  1. Load interview with participants
  2. Find participant matching userId
  3. If not a participant, throw 403
  4. Same LiveKit JWT generation as above
  5. Same lazy room creation
  6. Return same shape

**Add routes to routes.ts:**
- `POST /api/v2/interviews/join` — magic link exchange. Body: `{ token: string }`. NO AUTH REQUIRED (this is the whole point of magic links). Returns `{ data: { jwt, room_name, interview, participant } }`.
- `POST /api/v2/interviews/:id/token` — authenticated token generation. Requires x-clerk-user-id. Returns same shape as join.

**IMPORTANT:** The `/join` endpoint must NOT require the x-clerk-user-id header. Candidates use magic links without any authentication. The route handler should NOT call any auth middleware or check for clerk user ID.

**LiveKit SDK usage notes:**
- Import `AccessToken` from `livekit-server-sdk`
- The SDK is the official Node.js server SDK for LiveKit
- Read LIVEKIT_API_KEY and LIVEKIT_API_SECRET from environment (passed via constructor)
- Room name format: `interview-{interview.id}` (deterministic, idempotent)

**Token never expires per context decisions:** The magic link token in the DB has no expiry column — it's valid as long as the interview isn't cancelled/completed/expired. This is handled by checking interview status, not token timestamps.
