    Create a new `packages/shared-charts/` package following existing package patterns (look at shared-ui for reference):

    1. **package.json**: Name `@splits-network/shared-charts`, add dependencies:
       - `echarts` (latest stable)
       - `echarts-for-react` (latest stable)
       - Peer deps: `react`, `react-dom`
       - Copy tsconfig/build patterns from shared-ui

    2. **tsconfig.json**: Extend base tsconfig, include `src/`, jsx: react-jsx

    3. **src/theme.ts**: Create an ECharts theme object that reads DaisyUI CSS custom properties at runtime:
       - Map `--p` (primary), `--s` (secondary), `--a` (accent), `--su` (success), `--wa` (warning), `--er` (error) to ECharts color palette
       - Provide a `getThemeColors()` function that returns resolved CSS variable values
       - Provide a `splitsTheme` ECharts theme registration object
       - Use `oklch()` parsing since DaisyUI v5 uses oklch color format — read from `getComputedStyle(document.documentElement)`

    4. **src/hooks/use-echarts.ts**: Create `useECharts` hook that:
       - Registers the splits theme on mount
       - Returns `{ theme: 'splits', chartDefaults }` with standard options (responsive, animation config)
       - Handles SSR safety (typeof window check)

    5. **src/index.ts**: Barrel export (will be expanded in Task 2)

    6. **pnpm-workspace.yaml**: Verify `packages/*` glob already covers this (it should). If not, add it.

    Run `pnpm install` after creating the package to link workspace deps.
  </action>

  <action>
    Create chart components wrapping `echarts-for-react` with sensible defaults:

    1. **LineChart**: Props: `data: {x: string, y: number}[]`, `series?: multiple series`, `height?`, `showLegend?`, `xAxisLabel?`, `yAxisLabel?`, `smooth?`. Applies splits theme. Responsive.

    2. **BarChart**: Props: `data: {label: string, value: number}[]`, `horizontal?`, `stacked?`, `height?`. Applies splits theme.

    3. **PieChart**: Props: `data: {name: string, value: number}[]`, `donut?` (ring chart), `height?`, `showLabels?`. Applies splits theme.

    4. **AreaChart**: Same as LineChart but with `areaStyle` filled. Props: `gradient?` for gradient fill effect.

    5. **Sparkline**: Minimal inline chart. Props: `data: number[]`, `width?` (default 80), `height?` (default 24), `color?`, `type?: 'line' | 'bar'`. No axes, no grid, no tooltip — pure visual indicator for stat tiles. Keep under 60 lines.

    All components must:
    - Use `'use client'` directive
    - Apply splits theme via `useECharts` hook
    - Be responsive (opts.renderer = 'svg' for crisp rendering)
    - Handle empty data gracefully (show nothing or placeholder)
    - Stay under 200 lines each
    - Use `React.memo` for performance

    6. **Update index.ts** to export all components and hooks.

    7. **Add to admin app**: Add `"@splits-network/shared-charts": "workspace:*"` to `apps/admin/package.json` dependencies. Run `pnpm install`.
