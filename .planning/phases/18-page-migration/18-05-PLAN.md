  <action>
    1. **use-admin-stats.ts**: Hook that fetches aggregate stats from admin endpoints:
       - Calls `/admin/identity/admin/counts`, `/admin/ats/admin/counts`, `/admin/network/admin/counts`, `/admin/billing/admin/counts`, `/admin/notification/admin/counts` in parallel
       - Combines into unified stats object: `{ totalUsers, totalRecruiters, pendingRecruiters, totalJobs, totalApplications, totalCandidates, totalPlacements, totalAssignments, totalCompanies, totalMatches, pendingPayouts, activeEscrow, activeNotifications }`
       - Accepts `timePeriod` param for trend calculation (7d, 30d, 90d)
       - Includes `trend` data: percentage change vs prior period (if API supports, otherwise static for now)
       - Uses `useAuth()` for token, caches with react-query via `useQuery`
       - Returns `{ stats, loading, error, refetch }`

    2. **use-admin-activity.ts**: Hook for activity feed:
       - Fetches recent admin actions and platform activity
       - Two modes: 'admin' (admin-only actions) and 'all' (all platform activity)
       - Uses analytics or activity endpoints if available, otherwise stub with mock data
       - Returns `{ activities, loading, error, mode, setMode }`

    3. **dashboard-stats.tsx**: KPI stat tiles grid:
       - 6-8 stat tiles in responsive grid (3 cols lg, 2 cols md, 1 col mobile)
       - Each tile: icon, label, value (formatted number), trend indicator (up/down arrow + percentage), Sparkline chart
       - Tiles: Total Users, Active Recruiters (pending badge), Active Jobs, Applications (this period), Placements, Revenue (if billing data), Pending Payouts, Active Notifications
       - Basel design: card with subtle shadow, clean typography
       - Click on tile navigates to relevant detail page (using Next.js router)
       - Uses useAdminStats() hook
       - Uses Sparkline from shared-charts for mini trend lines

    4. **dashboard-actions.tsx**: Action items section:
       - Card listing pending items needing attention
       - Items: pending recruiter approvals (count + link), active fraud signals, pending payouts, expiring escrow holds
       - Each item: icon, description, count badge, "View" link
       - Uses counts from useAdminStats()
       - Basel card design

    5. **time-period-selector.tsx**: Global time period control:
       - DaisyUI select or button group: 7d, 30d, 90d, 1y, All
       - Lifted state passed down to chart components
       - Compact, sits in dashboard header area
  </action>
