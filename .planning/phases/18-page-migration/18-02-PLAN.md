  <action>
    Follow the analytics-gateway WebSocket pattern but simplified for admin:

    1. **package.json**: Add `ws` dependency (same version as analytics-gateway). Add `@types/ws` to devDeps.

    2. **src/realtime.ts**: Create `setupRealtimeServer(server: http.Server, authMiddleware: AdminAuthMiddleware, redis: Redis)`:
       - Create WebSocketServer attached to the HTTP server at path `/ws`
       - On connection: extract Bearer token from URL query param `?token=xxx`
       - Verify token using the same AdminAuthMiddleware pattern (Clerk verifyToken + isPlatformAdmin check)
       - Reject non-admin connections with close code 4001
       - Support channel subscriptions: client sends `{ type: 'subscribe', channels: ['counts', 'activity', 'health'] }`
       - Subscribe to Redis pub/sub for those channels (prefix: `admin:`)
       - Forward Redis messages to subscribed WebSocket clients
       - Heartbeat: ping every 30s, close stale connections after 60s no pong
       - Limit max 10 channels per socket
       - On disconnect: cleanup Redis subscriptions

    3. **src/index.ts**: After `app.listen()`, get the underlying HTTP server via `app.server` and call `setupRealtimeServer(app.server, authMiddleware, redis)`. Import the function.

    Run `pnpm install` in admin-gateway after adding deps.
  </action>
  <verify>
    `pnpm --filter @splits-network/admin-gateway build` compiles without errors.
    `grep "ws" services/admin-gateway/package.json` shows the dependency.
    realtime.ts exports setupRealtimeServer function.
  </verify>
  <done>Admin-gateway has WebSocket server at /ws with auth, channel subscriptions, and Redis pub/sub relay</done>

<task type="auto">

  <files>

  <action>
    1. **providers/realtime-provider.tsx**: Create `RealtimeProvider` context:
       - Uses `useAuth()` from Clerk to get token
       - Creates WebSocket connection to `${NEXT_PUBLIC_ADMIN_GATEWAY_URL}/ws?token={token}`
       - Replace https:// with wss:// and http:// with ws:// in the URL
       - Auto-reconnect with exponential backoff (1s, 2s, 4s, 8s, max 30s)
       - Expose context: `{ subscribe(channel: string, callback: (data: any) => void): () => void, connected: boolean }`
       - On mount: connect. On unmount: close.
       - Handle token refresh: if token changes, reconnect
       - `'use client'` directive

    2. **hooks/use-admin-realtime.ts**: Create `useAdminRealtime(channel: string, callback: (data: any) => void)`:
       - Subscribes to a channel via RealtimeProvider context
       - Sends subscribe message on mount
       - Returns `{ connected: boolean }`
       - Cleanup on unmount
       - `'use client'` directive

    3. **hooks/use-realtime-counts.ts**: Create `useRealtimeCounts()`:
       - Subscribes to 'counts' channel via useAdminRealtime
       - Returns `{ counts: Record<string, number>, loading: boolean }`
       - Initial load: fetches counts via REST API (GET /admin/identity/admin-counts or similar)
       - Updates counts in real-time as WebSocket messages arrive
       - Counts shape: `{ pendingRecruiters, activeFraud, pendingPayouts, activeEscrow, activeNotifications, totalUsers, totalJobs, totalApplications }`
       - Falls back to polling every 60s if WebSocket disconnects
       - `'use client'` directive
  </action>
