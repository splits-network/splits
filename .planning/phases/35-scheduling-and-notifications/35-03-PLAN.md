<task type="auto">
  <name>Task 1: Create scheduling service with slot computation and reschedule logic</name>
  <files>services/video-service/src/v2/interviews/scheduling-service.ts, services/video-service/src/v2/interviews/service.ts</files>
  <action>
1. **Create scheduling-service.ts:**
   - `SchedulingService` class with constructor taking `repository: InterviewRepository`, `eventPublisher: IEventPublisher`
   - `getAvailableSlots(params: { connectionId, clerkUserId, calendarIds, startDate, endDate, workingHoursStart, workingHoursEnd, workingDays, timezone, durationMinutes })`:
     - This method does NOT call the calendar API itself — it receives pre-fetched busy slots as input (the frontend or gateway orchestrates the calendar API call separately)
     - Actually, redesign: accept `busySlots: Array<{start: string, end: string}>` as parameter
     - Generate 30-minute increment slots within working hours for each working day in the date range
     - Filter out slots that overlap with any busy slot
     - Return `Array<{ start: string, end: string }>` — available time windows
     - All times in the user's timezone (use `Intl.DateTimeFormat` for timezone math)
   - `rescheduleInterview(interviewId, newScheduledAt, newDurationMinutes?, reason?, rescheduledByUserId?)`:
     - Validate interview exists and is in 'scheduled' status
     - Validate new time is in the future
     - Save original_scheduled_at if not already set (first reschedule)
     - Increment reschedule_count
     - Update scheduled_at and optionally scheduled_duration_minutes
     - Publish `interview.rescheduled` event with payload: `{interview_id, application_id, old_scheduled_at, new_scheduled_at, reason, rescheduled_by}`
     - Return updated interview
   - `createRescheduleRequest(interviewId, requestedBy, requestedByUserId, proposedTimes, notes?)`:
     - Validate interview exists and is in 'scheduled' status
     - Check reschedule_count < 2 if requested by candidate
     - Create reschedule request record
     - Publish `interview.reschedule_requested` event
     - Return the request
   - `acceptRescheduleRequest(requestId, acceptedTime)`:
     - Update request status to 'accepted', set accepted_time
     - Call rescheduleInterview with the accepted time
     - Publish `interview.reschedule_accepted` event
     - Return updated interview

2. **In service.ts:**
   - Update `createInterview` to accept and pass through new fields: `calendar_event_id`, `calendar_connection_id`, `meeting_platform`, `meeting_link`
   - When `meeting_platform` is not 'splits_video', still create the interview record but skip room_name generation (set to null) — room_name is only needed for LiveKit
   - Update the repository.createInterview call to include new fields
  </action>

  <name>Task 2: Add scheduling routes and gateway proxying</name>
  <files>services/video-service/src/v2/interviews/routes.ts, services/api-gateway/src/v2/routes.ts</files>

1. **In video-service routes.ts** — add new endpoints:

   - `PATCH /api/v2/interviews/:id/reschedule` — authenticated. Body: `{ scheduled_at, scheduled_duration_minutes?, reason? }`. Calls `schedulingService.rescheduleInterview`. Returns updated interview.

   - `POST /api/v2/interviews/:id/reschedule-request` — NO AUTH (candidate uses magic link token in body). Body: `{ token, proposed_times: [{start, end}], notes? }`. Validates token, checks reschedule_count < 2, calls `schedulingService.createRescheduleRequest`. Returns request.

   - `PATCH /api/v2/interviews/reschedule-requests/:requestId/accept` — authenticated. Body: `{ accepted_time }`. Calls `schedulingService.acceptRescheduleRequest`. Returns updated interview.

   - `POST /api/v2/interviews/available-slots` — authenticated. Body: `{ busy_slots: [{start, end}], start_date, end_date, working_hours_start, working_hours_end, working_days, timezone, duration_minutes }`. Calls `schedulingService.getAvailableSlots`. Returns available slots array.

   - `PATCH /api/v2/interviews/:id` — authenticated. Body: partial interview fields (for calendar event linking after creation). Calls `repository.updateInterview`. Returns updated interview.

   Instantiate SchedulingService in the route registration function, passing repository and eventPublisher.

   - The reschedule-request endpoint needs the no-auth bypass (same as /join endpoint)
   - The available-slots endpoint needs auth
