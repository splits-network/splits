<task type="auto">
  <name>Task 2: Create interview reminder scheduled job</name>
  <files>services/notification-service/src/jobs/send-interview-reminders.ts, services/notification-service/src/index.ts</files>
  <action>
1. **Create send-interview-reminders.ts:**
   Follow the pattern from `send-weekly-digest.ts` for job structure.

   - Export `sendInterviewReminders(supabaseClient, resendClient, logger, portalUrl, candidateWebsiteUrl)`:
     - Query `interviews` table for scheduled interviews where:
       - `status = 'scheduled'`
       - `scheduled_at` is within the next 24 hours + 5 minutes (covers all three reminder windows)
     - For each interview, determine which reminders to send:
       - Calculate time until interview: `scheduled_at - now()`
       - 24h reminder: send if between 23h55m and 24h5m (10-minute window to avoid duplicates)
       - 1h reminder: send if between 55min and 65min
       - 10min reminder: send if between 5min and 15min
     - **Deduplication**: Check notifications table for existing notification with `event_type = 'interview.reminder.{24h|1h|10min}'` and matching `payload.interview_id`. Skip if already sent.
     - For each reminder to send:
       - Fetch interview participants with user details (join interview_participants -> users)
       - Fetch application -> job -> company for context
       - Fetch access tokens for join links
       - For each participant:
         - Render appropriate reminder email (candidate vs interviewer variant)
         - Send via Resend
         - Create in-app notification:
           - event_type: `interview.reminder.{24h|1h|10min}`
           - subject: countdown text ("Interview in 55 minutes", "Interview starting now" for 10min)
           - action_url: interview page or application page
           - priority: 'normal' for 24h, 'high' for 1h, 'urgent' for 10min
           - payload: `{ interview_id, scheduled_at, reminder_type }`

2. **In index.ts:**
   - Import `sendInterviewReminders`
   - Add a cron job that runs every 5 minutes: `*/5 * * * *`
   - Follow the existing cron setup pattern used by other scheduled jobs
   - Pass required dependencies (supabaseClient, resendClient, logger, portalUrl, candidateWebsiteUrl)

The 5-minute interval ensures:
- Reminders are sent within a 10-minute accuracy window
- Each reminder type has a 10-minute detection window, avoiding duplicates
- The deduplication check prevents re-sending if the job runs multiple times within a window
  </action>
