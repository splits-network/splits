1. **Create interviewer-emails.ts** — email templates for the interviewer:

   - `interviewScheduledEmail(params: { interviewerName, candidateName, jobTitle, companyName, scheduledAt, duration, meetingPlatform, joinLink?, applicationUrl, resumeUrl? })`:
     - Subject: "Interview Scheduled: [Candidate Name] for [Job Title]"
     - Body: date/time, candidate name, job title, meeting platform, "View Application" CTA button linking to application page, resume link if available
     - Use base template wrapper

   - `interviewCancelledEmail(params: { interviewerName, candidateName, jobTitle, scheduledAt, reason? })`:
     - Subject: "Interview Cancelled: [Candidate Name] for [Job Title]"
     - Body: cancelled notice, original date/time, reason (if provided), "View Application" link

   - `interviewRescheduledEmail(params: { interviewerName, candidateName, jobTitle, oldScheduledAt, newScheduledAt, duration, reason?, joinLink? })`:
     - Subject: "Interview Rescheduled: [Candidate Name] for [Job Title]"
     - Body: old time (struck through), new time (highlighted), reason, updated join link

   - `rescheduleRequestedEmail(params: { interviewerName, candidateName, jobTitle, proposedTimes: string[], notes?, actionUrl })`:
     - Subject: "[Candidate Name] requested to reschedule interview"
     - Body: candidate's proposed times, notes, "Review Request" CTA button

2. **Create candidate-emails.ts** — email templates for the candidate:

   - `candidateInterviewScheduledEmail(params: { candidateName, jobTitle, companyName, interviewerName, scheduledAt, duration, meetingPlatform, joinLink?, prepPageUrl, rescheduleLink? })`:
     - Subject: "Your Interview: [Job Title] at [Company Name]"
     - Body: rich prep info — date/time in candidate's timezone context, job title, company name, interviewer name, meeting platform, "Join Interview" CTA button (or meeting link), "Prepare for Interview" link to prep page, reschedule link
     - Most detail-rich email of all

   - `candidateInterviewCancelledEmail(params: { candidateName, jobTitle, companyName, scheduledAt, reason? })`:
     - Subject: "Interview Cancelled: [Job Title] at [Company Name]"
     - Body: cancellation notice, reason if provided

   - `candidateInterviewRescheduledEmail(params: { candidateName, jobTitle, companyName, oldScheduledAt, newScheduledAt, duration, joinLink?, prepPageUrl })`:
     - Subject: "Interview Rescheduled: [Job Title] at [Company Name]"
     - Body: old time, new time, updated details, prep page link

   - `candidateRescheduleAcceptedEmail(params: { candidateName, jobTitle, companyName, newScheduledAt, duration, joinLink?, prepPageUrl })`:
     - Subject: "Reschedule Confirmed: [Job Title] at [Company Name]"
     - Body: confirmed new time, all details refreshed

3. **Create index.ts** — barrel export for the templates directory.

<task type="auto">
  <name>Task 2: Create interview event consumer and wire into domain consumer</name>
  <files>services/notification-service/src/consumers/interviews/consumer.ts, services/notification-service/src/services/interviews/service.ts, services/notification-service/src/domain-consumer.ts, services/notification-service/src/service.ts</files>
  <action>
1. **Create services/interviews/service.ts** — interview email service:
   - `InterviewsEmailService` class with constructor taking `resendClient`, `logger`, `portalUrl`, `candidateWebsiteUrl`
   - Methods that render templates and send via Resend:
     - `sendInterviewScheduled(params)` — sends to both interviewer and candidate
     - `sendInterviewCancelled(params)` — sends to both
     - `sendInterviewRescheduled(params)` — sends to both
     - `sendRescheduleRequested(params)` — sends to interviewer only
     - `sendRescheduleAccepted(params)` — sends to candidate only
   - Follow pattern from existing email services (e.g., `services/applications/service.ts`)

2. **Create consumers/interviews/consumer.ts** — event consumer:
   - `InterviewsEventConsumer` class following existing pattern (see applications consumer)
   - Constructor: `emailService`, `logger`, `portalUrl`, `candidateWebsiteUrl`, `dataLookup`, `contactLookup`
   - Event handlers:

     `handleInterviewCreated(event)`:
     - Extract: interview_id, application_id, scheduled_at, created_by, participant_count
     - Lookup: application (for job_id, candidate_id), job (for title, company), participants (for user details)
     - Get candidate email via contactLookup
     - Get interviewer email via contactLookup
     - Build prep page URL: `${candidateWebsiteUrl}/interview/{accessToken}` (need to lookup access token)
     - Build application URL: `${portalUrl}/portal/applications/{application_id}`
     - Build join link based on meeting_platform
     - Send confirmation emails to all participants
     - Create in-app notification for interviewer: event_type `interview.scheduled`, action_url to application page, category `interviews`

     `handleInterviewCancelled(event)`:
     - Extract: interview_id, application_id, reason, cancelled_by
     - Lookup same data as above
     - Send cancellation emails to all participants
     - Create in-app notification: event_type `interview.cancelled`, category `interviews`

     `handleInterviewRescheduled(event)`:
     - Extract: interview_id, old_scheduled_at, new_scheduled_at, reason, rescheduled_by
     - Lookup same data
     - Send reschedule emails to all participants
     - Create in-app notification: event_type `interview.rescheduled`, category `interviews`

     `handleRescheduleRequested(event)`:
     - Extract: interview_id, proposed_times, requested_by
     - Send reschedule request email to interviewer
     - Create in-app notification for interviewer

     `handleRescheduleAccepted(event)`:
     - Extract: interview_id, new_scheduled_at
     - Send acceptance email to candidate
     - Create in-app notification for candidate (via in-app notification to their user_id)

3. **In domain-consumer.ts:**
   - Import InterviewsEventConsumer
   - Add `interviewsConsumer` property
   - Instantiate in constructor with same pattern as other consumers
   - Bind to events: `interview.created`, `interview.cancelled`, `interview.rescheduled`, `interview.reschedule_requested`, `interview.reschedule_accepted`
   - Add switch cases in handleEvent for each event type

4. **In service.ts (NotificationService):**
   - Add `interviews: InterviewsEmailService` property
   - Instantiate in constructor
