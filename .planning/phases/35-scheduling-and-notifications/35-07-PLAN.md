  <files>apps/portal/src/components/basel/scheduling/reschedule-interview-modal.tsx, apps/portal/src/components/basel/scheduling/cancel-interview-dialog.tsx</files>
  <action>
1. **Create reschedule-interview-modal.tsx:**
   - Props: `interviewId: string`, `currentScheduledAt: string`, `currentDuration: number`, `candidateName: string`, `jobTitle: string`, `calendarEventId?: string`, `calendarConnectionId?: string`, `onClose: () => void`, `onSuccess: () => void`
   - Uses BaselWizardModal with 2 steps: "New Time" and "Confirm"
   - Step 1 "New Time":
     - Show current time in info box: "Currently scheduled: [formatted date/time]"
     - Date picker pre-filled with current date
     - Time picker pre-filled with current time
     - Duration dropdown pre-filled with current duration
     - If calendar connected: fetch and show available slots via AvailableSlotsList
     - Reason textarea: "Reason for reschedule (optional)"
   - Step 2 "Confirm":
     - Review card showing old time -> new time comparison
     - Duration, reason
     - "Participants will be notified of the change" info note
   - On submit:
     1. PATCH `/interviews/{id}/reschedule` with `{ scheduled_at, scheduled_duration_minutes, reason }`
     2. If `calendarEventId && calendarConnectionId`: PATCH `/integrations/calendar/{connectionId}/events/{eventId}` with new start/end times
     3. Call onSuccess, close modal

2. **Create cancel-interview-dialog.tsx:**
   - Props: `interviewId: string`, `candidateName: string`, `scheduledAt: string`, `calendarEventId?: string`, `calendarConnectionId?: string`, `onClose: () => void`, `onSuccess: () => void`
   - Uses ModalPortal + DaisyUI modal (not wizard — single step)
   - Content:
     - Warning icon and "Cancel Interview" header
     - Info text: "Are you sure you want to cancel the interview with [candidateName] scheduled for [formatted date]?"
     - Reason textarea: "Reason for cancellation (optional)" — included in notifications
     - "All participants will be notified" info note
   - Buttons: "Keep Interview" (ghost) and "Cancel Interview" (btn-error)
   - On confirm:
     1. PATCH `/interviews/{id}/cancel` with `{ reason }`
     2. If `calendarEventId && calendarConnectionId`: DELETE `/integrations/calendar/{connectionId}/events/{eventId}`
     3. Call onSuccess, close dialog

Both components use DaisyUI classes only, no rounded corners, minimum `text-sm` text.
