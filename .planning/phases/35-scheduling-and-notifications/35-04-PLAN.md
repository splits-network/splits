  <action>
Rewrite the existing ScheduleInterviewModal with the following changes while keeping the BaselWizardModal wrapper:

**Props changes:**
- Add `applicationId: string` (required)
- Add `applicationStage?: string` — if terminal state, show error and block
- Keep existing: `candidateName`, `candidateEmail`, `jobTitle`, `onClose`, `onSuccess`

**Steps update** (3 steps remain but content changes):
1. "Platform & Calendar" — platform selector + calendar selection (if not DB-only)
2. "Date & Time" — date picker + available slots list (if calendar connected) or manual time entry (if DB-only)
3. "Confirm" — review details

**Step 1 behavior:**
- Render `PlatformSelector` at top
- Below: existing calendar/connection selector (only shown if platform !== 'splits_video' OR user has calendar connected for event creation)
- Support "no calendar" mode: if user has no calendar connections, show info that event will be created with .ics file attachment only
- Store selected platform in state

**Step 2 behavior:**
- If calendar connected: fetch free/busy data via `POST /integrations/calendar/{connectionId}/availability` for primary calendar, next 2 weeks
- Pass busy slots to `POST /interviews/available-slots` to get computed available slots
- Render `AvailableSlotslist` component
- Also show manual date/time entry below as alternative ("Or pick a custom time")
- Duration dropdown default: 30 minutes (changed from 60)
- Remove title input — auto-generate: `Interview: ${candidateName} — ${jobTitle}`

**Step 3 behavior:**
- Show auto-generated title (non-editable, displayed as text)
- Show selected platform with icon
- Show date/time, duration
- Show candidate email (auto-added as attendee)
- Show additional attendee emails input
- Auto-generated description: "Interview for [Job Title] position\nCandidate: [Name]\nApplication: [portal link]"

**Submit behavior:**
1. If calendar connected: create calendar event via `POST /integrations/calendar/{connectionId}/events`
2. If platform is 'splits_video': also create interview via `POST /interviews` with `calendar_event_id` from step 1 (if calendar was used), `meeting_platform: 'splits_video'`
3. If platform is 'google_meet' or 'microsoft_teams': only create calendar event (no interviews DB record per CONTEXT.md decision)
4. If no calendar: create interview via `POST /interviews` with `meeting_platform: 'splits_video'` and no calendar_event_id
5. Publish success, call onSuccess callback

**Terminal state blocking:**
- If applicationStage is 'rejected', 'hired', or 'withdrawn': show BaselAlertBox error "Cannot schedule interview for an application in [stage] state" and disable next/submit buttons
