  <name>Task 1: Candidate reschedule request flow</name>
  <files>apps/candidate/src/app/interview/[token]/components/reschedule-request-form.tsx, apps/candidate/src/app/interview/[token]/components/prep-page.tsx</files>
  <action>
1. **Create reschedule-request-form.tsx:**
   - Props: `interviewId: string`, `token: string`, `interviewerAvailableSlots: Array<{start: string, end: string}>`, `candidateTimezone: string`, `rescheduleCount: number`, `maxReschedules: number`, `onSuccess: () => void`
   - If `rescheduleCount >= maxReschedules` (2): show info "You've reached the maximum number of reschedule requests" — disable form
   - If reschedule already pending (check via API): show "Reschedule request pending — waiting for interviewer response" with badge "Reschedule requested"

   **Slot selection UI:**
   - Show available slots for next 2 weeks in 30-min increments, grouped by day
   - Candidate selects 2-3 preferred time slots (toggle buttons)
   - Slots shown in candidate's local timezone (auto-detected from browser)
   - Minimum 2 slots, maximum 3 slots required

   **Notes field:**
   - Textarea: "Reason for reschedule (optional)"

   **Submit:**
   - POST to `/interviews/{interviewId}/reschedule-request` with `{ token, proposed_times: [{start, end}], notes }`
   - No auth needed — token-based (magic link candidate)
   - On success: show success message "Reschedule request sent to interviewer"
   - On error: show error message

   - DaisyUI classes, no rounded corners, minimum `text-sm`
   - Selected slots: `border-primary bg-primary/5`

2. **Update prep-page.tsx:**
   - Fetch available slots for the interviewer:
     - Need to get interviewer's calendar availability. The candidate doesn't have calendar access.
     - Approach: add a new endpoint `GET /api/v2/interviews/{id}/available-slots?token={token}` in video-service that:
       1. Validates the magic link token
       2. Looks up the interviewer's calendar preferences (connection_id, calendar_ids, working hours)
       3. Fetches free/busy from integration-service calendar API (service-to-service call or direct DB)
       4. Computes available slots for next 2 weeks
       5. Returns slots array
     - Add this endpoint in video-service routes (no auth, token-validated)
     - Add gateway proxy for this endpoint
   - Add "Need to reschedule?" section below the interview details:
     - Show only if interview status is 'scheduled' and reschedule_count < 2
     - Collapsible section (DaisyUI collapse): click to expand reschedule form
     - When expanded, fetch available slots and render RescheduleRequestForm
   - If reschedule already requested: show "Reschedule requested" badge on the prep page
  </action>

   - Renders as a compact inline element within the notification card
   - When < 10 minutes: show "Join Now" button inline (`btn btn-primary btn-sm`)

     - Interview icon `fa-video` instead of default bell icon
     - Category badge "Interview" with `badge badge-accent`
     - Countdown component
     - "Join Now" action when within 10 minutes
