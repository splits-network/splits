  <action>
Mirror the candidate-skills module (`services/ats-service/src/v2/candidate-skills/`) with company_id instead of candidate_id:

**repository.ts — `CompanySkillRepository`:**
- Constructor takes `SupabaseClient` (injected, same as CandidateSkillRepository)
- `listByCompanyId(companyId)` — query `company_skills` table with `.select('*, skill:skills(*)').eq('company_id', companyId).order('created_at', { ascending: true })`. Returns junction rows with joined skill data
- `bulkReplace(companyId, skills: Array<{ skill_id: string }>)` — delete all `.eq('company_id', companyId)`, then insert new rows `{ company_id, skill_id }`, select with `*, skill:skills(*)`. Return empty array if skills is empty (after delete). Same delete-then-insert pattern as candidate-skills

**service.ts — `CompanySkillService`:**
- Constructor takes `CompanySkillRepository`
- `listByCompanyId(companyId)` — validate companyId, delegate to repository
- `bulkReplace(companyId, skills)` — validate companyId, validate skills is array, validate each has skill_id, enforce max 50 items (`if (skills.length > 50) throw new Error('Maximum 50 skills per company')`), delegate to repository

**routes.ts — `registerCompanySkillRoutes`:**
- `GET /api/v2/company-skills` — requireUserContext, parse `company_id` from query params (required), call `service.listByCompanyId(company_id)`, return `{ data }`
- `PUT /api/v2/companies/:companyId/skills` — requireUserContext, parse `companyId` from params, parse `{ skills: Array<{ skill_id: string }> }` from body, validate skills array exists, call `service.bulkReplace(companyId, skills)`, return `{ data }`
- Error handling: same try/catch as candidate-skills routes

NOTE: No individual add/delete endpoints — the CONTEXT decisions specify bulk-replace as the sole mutation mechanism. The UI will always send the complete set.
  </action>
