  <action>
Create three files mirroring the skills module (`services/ats-service/src/v2/skills/`) with these substitutions:

**repository.ts ‚Äî `CultureTagRepository`:**
- Constructor takes `supabaseUrl` and `supabaseKey`, creates Supabase client (same as SkillRepository)
- `getSupabase()` method exposing the client
- `search(query, limit=20)` ‚Äî query the `culture_tags` table with `.ilike('name', '%query%')`, order by name, limit. Min 1 char (return empty if less). Default limit 20
- `getById(id)` ‚Äî `.eq('id', id).maybeSingle()`, throw if not found
- `getBySlug(slug)` ‚Äî `.eq('slug', slug).maybeSingle()`, return null if not found
- `create(name, slug, createdBy?)` ‚Äî insert into `culture_tags`, select and return single

**service.ts ‚Äî `CultureTagService`:**
- Constructor takes `CultureTagRepository` and optional `SupabaseClient` (for AccessContextResolver, same as SkillService)
- `searchCultureTags(query, limit=20)` ‚Äî delegates to repository.search
- `findOrCreate(name, clerkUserId?)` ‚Äî trim name, generate slug using same `generateSlug` function (copy from skills service), check existing by slug, return if found. If not found, resolve clerkUserId via AccessContextResolver, then create
- `getCultureTag(id)` ‚Äî delegates to repository.getById

**routes.ts ‚Äî `registerCultureTagRoutes`:**
- `GET /api/v2/culture-tags` ‚Äî requireUserContext, parse `q` and `limit` from query. If `q` provided, call `service.searchCultureTags(q, limit)`. Return `{ data }`
- `POST /api/v2/culture-tags` ‚Äî requireUserContext, parse `{ name }` from body, call `service.findOrCreate(name, clerkUserId)`. Return 200 if slug existed, 201 if newly created, both with `{ data }`
- Error handling: same try/catch pattern as skills routes

NOTE: This is structurally identical to the perks module. The only differences are: table name (`culture_tags`), class names (`CultureTag*`), route paths (`/api/v2/culture-tags`), and method names (`searchCultureTags`, `getCultureTag`).
