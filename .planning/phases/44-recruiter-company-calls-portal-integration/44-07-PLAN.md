  <action>
**call-creation-modal.tsx** — DaisyUI modal (`modal modal-bottom sm:modal-middle`):
- Props: `isOpen`, `onClose`, `defaultParticipants?`, `defaultEntityType?`, `defaultEntityId?`, `defaultMode?: 'instant' | 'scheduled'`
- Sections:
  1. **Mode toggle**: "Call Now" vs "Schedule" tabs/buttons at top
  2. **Participant picker** component
  3. **Entity linker** component (optional, collapsible)
  4. **Scheduling panel** (only shown in "Schedule" mode) — separate plan task
  5. **Title** input (optional, auto-generated if empty: "Call with {participant names}")
  6. **Agenda** textarea (optional, shown in Schedule mode)
  7. **Pre-call notes** textarea (optional, private to scheduler)
  8. **Tags** picker (optional)
  9. **Duration** select: 15/30/45/60 min (Schedule mode only)
  10. **Confirmation step for instant**: "Call {Name}? They will be notified." with Call/Cancel buttons
- Submit: calls `useCreateCall` hook
- After instant call creation: redirect to video app with magic link token (call POST /api/v2/calls/:id/token to get access token, then redirect to video.splits.network/join?token=xxx)
- After scheduled call creation: show success toast, close modal

**participant-picker.tsx:**
- Search input that queries existing users (portal users — both recruiters and company contacts)
- Uses existing user search endpoint or `/api/v2/search?type=users&q=`
- Displays results as cards with avatar, name, email, role, online/offline status (using existing presence components)
- Selected participants shown as chips/tags above search
- "Add by email" option: if search returns no results, show "Invite {email}" option
- Multi-participant support (add multiple people)
- Creator is auto-added as participant (shown but not removable)

**entity-linker.tsx:**
- If `defaultEntityType` and `defaultEntityId` provided: pre-filled and shown as a chip
- Otherwise: two-step picker:
  1. Entity type select: Company or Job
  2. Search for entity by name (company search or job search endpoint)
- Optional — can be left empty (especially for global/chat-initiated calls)
- Show linked entity as a card with name, type badge

**use-create-call.ts:**
- `useCreateCall()` hook returning `{ createCall, isLoading, error }`
- `createCall(input)` → POST `/api/v2/calls` with full input
- For instant calls: after creation, POST `/api/v2/calls/:id/token` to get access token
- Returns: `{ call, accessToken? }` (accessToken only for instant calls)

**index.ts:** Barrel export for all call creation components.

**scheduling-panel.tsx:**
- Fetches calendar availability from `/api/v2/calendar/availability` for selected participants
- If ALL participants have Google Calendar connected:
  - Show slot-based picker: date selector + available time slots as buttons
  - Slots are computed by finding gaps in all participants' busy_slots
  - Slots respect duration selection (only show slots with enough contiguous free time)
  - Selected slot highlighted with `bg-primary text-primary-content`
- If ANY participant has no Google Calendar:
  - Fall back to free-form date/time picker
  - DaisyUI date input + time input
  - Show note: "{name} doesn't have Google Calendar connected — choose a time manually"
- Shows selected date/time summary below the picker
- Time zone display: show user's local timezone, clarify if participants are in different zones

**tag-picker.tsx:**
- Fetches available tags from `/api/v2/calls/tags`
- Renders as horizontal row of DaisyUI toggle buttons/checkboxes
- Tags: intro, check-in, negotiation, follow-up, onboarding, review
- Multi-select: toggle on/off
- Selected tags shown with `badge badge-primary` styling
- Compact — single row with wrap on mobile

Both components use DaisyUI form controls exclusively. No custom date pickers — use native HTML date/time inputs styled with DaisyUI input classes, or the existing date picker pattern if one exists in the portal.
