**Create `scheduler.ts`:**

```
export class CallScheduler {
  constructor(
    private repository: CallRepository,
    private eventPublisher: IEventPublisher,
    private intervalMs: number = 60_000, // 1-minute check interval
  ) {}

  private timer: NodeJS.Timeout | null = null;

  start() { this.timer = setInterval(() => this.tick(), this.intervalMs); }
  stop() { if (this.timer) clearInterval(this.timer); }

  private async tick() {
    try {
      await Promise.all([
        this.processReminders(),
        this.processInstantCallTimeouts(),
        this.processNoShows(),
      ]);
    } catch (err) {
      console.error('[CallScheduler] tick error:', err);
    }
  }
}
```

**`processReminders()`:**
- For each reminder type ('24h', '1h', '5min'):
  - Query getCallsNeedingReminder
  - For each call, publish event:
    - '24h' and '1h': `call.reminder` event with { call_id, reminder_type, scheduled_at, participants }
    - '5min': `call.starting_soon` event (separate event for in-app toast handling)
  - Mark reminder sent

**`processInstantCallTimeouts()`:**
- Query getTimedOutInstantCalls
- For each, call markCallMissed and publish `call.missed` event

**`processNoShows()`:**
- Query getNoShowScheduledCalls
- For each, call markCallNoShow and publish `call.no_show` event with participant info

**In `index.ts`:**
- After registering routes and creating the repository/eventPublisher, create and start the scheduler:
  ```
  const scheduler = new CallScheduler(repository, eventPublisher);
  scheduler.start();
  app.addHook('onClose', () => scheduler.stop());
  ```
- The scheduler runs within the call-service process (no separate cron service needed for MVP)

- `call.missed` → no email (missed calls are just a status, per CONTEXT.md)
- `call.no_show` → send no-show notification to both participants (add a simple no-show template or reuse cancellation with different subject)
