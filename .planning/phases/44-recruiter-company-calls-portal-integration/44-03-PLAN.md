  <name>Task 1: Create call notification consumer and service</name>
  <files>
    services/notification-service/src/consumers/calls/consumer.ts

  <action>
**Follow the interview notification pattern exactly** — look at `consumers/interviews/` and `services/interviews/service.ts` as the template.

**Create `consumers/calls/consumer.ts`:**
- Subscribe to RabbitMQ exchange for call events
- Route events to the call notification service:
  - `call.created` → if `scheduled_at` exists: sendConfirmation, else: sendInstantCallNotification
  - `call.cancelled` → sendCancellation
  - `call.rescheduled` → sendRescheduleNotification
  - `call.recording.ready` → sendRecordingReady
- Export a `registerCallConsumer` function matching the interview consumer pattern
- Handle errors with try/catch + logging, do not let consumer crash

**Create `services/calls/service.ts`:**
- Class `CallNotificationService` with constructor accepting Resend client and DB/repository access (follow interview service pattern)
- Methods:
  1. `sendConfirmation(event)` — send to ALL participants. Include: date/time + timezone, participant names, entity context (company/job name), agenda (if provided), "Join Call" button URL (magic link from portal), "View in Portal" link. Brand-aware (check entity links for brand detection).
  2. `sendInstantCallNotification(event)` — send to all participants EXCEPT creator. Include: caller name, entity context, "Accept" button (join URL), urgency styling. Brand-aware.
  3. `sendCancellation(event)` — send to all participants. Include: who cancelled, optional reason, entity context. Brand-aware.
  4. `sendRescheduleNotification(event)` — send to all participants. Include: new date/time, entity context, updated "Join Call" button. No confirmation step needed. Brand-aware.
  5. `sendRecordingReady(event)` — send to all participants. Include: call title, date, "View Recording" link to portal detail page. No summary preview in email. Brand-aware.

- Each method:
  - Fetches participant details (names, emails) from users table
  - Fetches entity link details (company name, job title) for context
  - Determines brand from entity links (applicant.network if candidate entities, splits.network otherwise)
  - Renders the appropriate email template
  - Sends via the existing Resend client pattern

**Register the consumer** in `domain-consumer.ts` — add import and call to `registerCallConsumer` alongside existing consumers.
  </action>
  <verify>Run `pnpm --filter @splits-network/notification-service build` to verify TypeScript compiles.</verify>
  <done>Consumer routes all call events to notification service. Service resolves participants, entities, and brands for each notification type.</done>
</task>

Follow the existing interview email template pattern in `templates/interviews/`. Use the shared `base.ts` layout wrapper and `components.ts` for buttons, headers, footers.

**Create 6 templates:**

1. **confirmation.ts** — "Your call is scheduled"
   - Subject: "Call Scheduled: {title or 'Call with {participant_names}'}"
   - Body: date/time with timezone, participant list, entity context (company/job), agenda text (if provided), two CTAs: "Join Call" (primary) + "View in Portal" (secondary)

2. **reminder.ts** — "Your call is coming up"
   - Subject: "Reminder: Call with {participant_names} in {time_until}"
   - Body: simplified — participant names, date/time, single "Join Call" button
   - Used for both 24h and 1h reminders (time_until param varies)

3. **cancellation.ts** — "Call cancelled"
   - Subject: "Call Cancelled: {title or participant_names}"
   - Body: who cancelled, reason (if provided), entity context, original date/time

4. **reschedule.ts** — "Call rescheduled"
   - Subject: "Call Rescheduled: {title or participant_names}"
   - Body: new date/time with timezone, entity context, "Join Call" button with updated URL

5. **recording-ready.ts** — "Your recording is ready"
   - Subject: "Recording Ready: {title or participant_names}"
   - Body: call date, duration, "View Recording" button linking to portal detail page. No summary preview.

6. **instant-call.ts** — "Incoming call"
   - Subject: "{caller_name} is calling you"
   - Body: caller name, entity context (if any), prominent "Join Call" button, urgency styling (bold subject, highlight border)

**Create index.ts** barrel export for all templates.

All templates must:
- Accept brand parameter for logo/colors
- Use the shared base layout
- Include proper unsubscribe footer
- Have both HTML and plain text versions (follow existing pattern)
