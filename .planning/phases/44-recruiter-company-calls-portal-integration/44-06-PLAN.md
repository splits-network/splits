- Horizontal bar with 5 stat cards: Upcoming, This Week, This Month, Avg Duration, Needs Follow-up
- Use `bg-base-200` cards with `text-primary` for values
- DaisyUI stat component pattern if available, otherwise simple card grid
- `grid-cols-2 md:grid-cols-3 lg:grid-cols-5` responsive

**call-filters.tsx:**

- Filter dropdowns using DaisyUI select/dropdown:
    - Call Type: dropdown with call_type options (interview, client_meeting, etc.)
    - Status: upcoming, completed, cancelled, missed, no_show (multi-select or single)
    - Date range: date inputs for from/to
    - Entity: entity type + entity name search (auto-complete or simple text)
    - Tags: multi-select from available tags list
    - Needs Follow-up: toggle/checkbox
- Compact filter bar that expands on mobile
- "Clear filters" button

**controls-bar.tsx:**

- Search input (text-sm, DaisyUI input), view mode toggle (table/grid icons), "New Call" button (primary)
- Search debounced (300ms)

**call-table.tsx + call-row.tsx:**

- Columns: Title/Participants, Type, Status, Date/Time, Duration, Entity, Tags, Actions
- Status uses `badge` with semantic colors (badge-success for completed, badge-warning for upcoming, badge-error for missed/no_show, badge-primary for cancelled)

- Tags: small badges inline
- Participant avatars: stacked avatar group (first 3 + "+N" overflow)
- Date: formatted with relative time for recent ("2h ago"), absolute for older
- Actions column: "View" link to detail page, "Join" button if upcoming and within 5min of scheduled_at
- Column hiding on mobile: hide Duration, Tags, Entity columns (`hidden md:table-cell`)
- Sortable column headers (click to sort by that column)
- Needs follow-up: small flag icon or dot indicator

**call-grid.tsx + call-card.tsx:**

- Card with: participant avatars at top, title, date/time, status badge, entity badge, tags row
