**recording-tab.tsx:**
- Video/audio player using HTML5 `<video>` or `<audio>` element
- Accept `currentTimestamp` from parent — when timestamp changes, seek to it
- Show recording status: if `recording_status !== 'ready'`, show processing indicator ("Recording processing..." with DaisyUI loading spinner)
- Playback controls: play/pause, seek, volume, playback speed (1x, 1.5x, 2x)
- Duration display

**transcript-tab.tsx:**
- Render transcript as timestamped lines
- Each line: `[HH:MM:SS] Speaker: text`
- Clicking a timestamp calls `setCurrentTimestamp(seconds)` which syncs the recording player
- Active line highlighted (the line closest to current playback position)
- Auto-scroll to active line during playback
- If `transcript_status !== 'ready'`: show "Transcript processing..." indicator
- Searchable: text input at top to filter transcript lines

**summary-tab.tsx:**
- Render structured AI summary (JSON from call_summaries.summary field)
- Expected sections: Key Points, Action Items, Decisions Made, Follow-ups
- Action items rendered as checkbox list (clicking converts to follow-up tracking — future, for now just display)
- If `summary_status !== 'ready'`: show "Summary generating..." indicator
- Clean typography with proper heading hierarchy

**call-context-panel.tsx:**
- Shows entity context based on entity_links:
  - Company: name, logo, industry, size, active job count, contacts list
  - Job: title, company, location, salary range, status
- If no entity linked: show participant profiles with name, role, email
- If call has agenda: show agenda text (editable only by creator if call is upcoming)
- Read-only entity info (no editing)
- Collapsible on mobile

**call-notes-section.tsx:**
- List of existing notes with author name and timestamp
- "Add Note" textarea + submit button at bottom
- POST to `/api/v2/calls/:id/notes` (need to add this route if not exists — check call-service routes)
- Notes are participant reflections after the call
- Private visibility (only the note author sees their notes — Claude's discretion: making them private)

**call-participants-section.tsx:**
- Table or card list of participants
- Columns: Name, Role, Joined At, Left At, Duration (computed), Consent status
- Online/offline status using presence components

**related-calls-section.tsx:**
- Shows up to 5 related calls (same entity or same participants)
- Simple card list with: title, date, status badge, link to detail
- "View all" link to calls list with entity filter pre-applied
