**use-call-context.ts:**
- Accepts `callId` and `accessToken` (magic-link token for auth)
- Fetches call detail from gateway: `/api/v2/calls/{callId}?include=entity_links,participants`
- For each entity link, fetches entity data:
  - Company: `/api/v2/companies/{entityId}` — name, logo, industry, size, contacts
  - Job: `/api/v2/jobs/{entityId}` — title, company, location, salary, status
- Returns: `{ entities, participants, agenda, isGuest, isLoading }`
- `isGuest` determined by whether the access token was magic-link (no Clerk user) — check based on participant data

**use-panel-preference.ts:**
- Stores last active tab in localStorage keyed by callId: `panel-tab-{callId}`
- `getLastTab(callId)` returns saved tab or default 'context'
- `saveTab(callId, tab)` persists the preference

**context-tab.tsx:**
- Renders entity data fetched by use-call-context:
  - **Company section**: logo + name header, industry badge, company size, active job count, contacts list (name + role + email)
  - **Job section** (if job-linked): title, company name, location, salary range, status badge, key stats
- If call has agenda: show in a highlighted card (`bg-base-200 border-l-4 border-primary`)
  - If viewer is the scheduler: agenda is editable (inline edit, PUT to update)
  - If viewer is not scheduler: read-only
- Pre-call notes: shown ONLY to the scheduler (check created_by match)
- If no entity linked: show prompt "Link this call to a company or job" with entity picker (reuse entity-linker concept, simplified for video app)
- Falls back to participant profile cards (avatar, name, role) if no entity and user declines to link

**history-tab.tsx:**
- Shows recent interactions with the same participants or entity (last 5-10)
- Fetch from: `/api/v2/calls?entity_id={entityId}&limit=10&sort_order=desc` (if entity linked)
- Or: list calls with same participants (future — for now, entity-based only)
- Each interaction: compact card with date, type, title, duration, status
- "View all" link opens portal call list (in new tab) with entity filter
- If no entity: show "No history available" with note about linking entity
