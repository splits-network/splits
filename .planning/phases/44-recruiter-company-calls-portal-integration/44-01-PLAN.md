<task type="auto">
  <name>Task 1: Create migration for scheduling fields, tags, and expanded statuses</name>
  <files>services/call-service/migrations/20260308000001_call_scheduling_tags.sql</files>
  <action>
Create a migration that:

1. **Expand call statuses** - ALTER the CHECK constraint on `calls.status` to add `'missed'` and `'no_show'` values. The existing constraint allows `scheduled`, `active`, `completed`, `cancelled`. Drop and recreate the constraint.

2. **Add scheduling columns to `calls` table:**
   - `agenda TEXT` — optional agenda/notes for the call (included in confirmation email)
   - `duration_minutes_planned INTEGER` — selected duration (15/30/45/60), distinct from actual `duration_minutes`
   - `pre_call_notes TEXT` — private notes for the scheduler (visible only to them)
   - `needs_follow_up BOOLEAN DEFAULT false` — manual follow-up flag
   - `cancelled_by TEXT REFERENCES users(id)` — who cancelled (for cancel notifications)
   - `cancel_reason TEXT` — optional cancellation reason

3. **Create `call_tags` lookup table:**
   - `slug TEXT PRIMARY KEY` — e.g., 'intro', 'check-in', 'negotiation'
   - `label TEXT NOT NULL`
   - `created_at TIMESTAMPTZ DEFAULT NOW()`
   - Seed with default tags: intro, check-in, negotiation, follow-up, onboarding, review

4. **Create `call_tag_links` junction table:**
   - `id UUID PRIMARY KEY DEFAULT gen_random_uuid()`
   - `call_id UUID NOT NULL REFERENCES calls(id) ON DELETE CASCADE`
   - `tag_slug TEXT NOT NULL REFERENCES call_tags(slug) ON DELETE CASCADE`
   - `created_at TIMESTAMPTZ DEFAULT NOW()`
   - UNIQUE constraint on (call_id, tag_slug)
   - Index on call_id for join performance

5. **Add RLS policies** for call_tags (read-only for authenticated) and call_tag_links (same as calls table pattern — participant-based).

Use BEGIN/COMMIT transaction wrapper. Follow existing migration naming pattern.
  </action>
