<task type="auto">
  <name>Task 2: Fix recording playback to use signed URL from playback-url endpoint</name>
  <files>
    apps/portal/src/app/portal/calls/[id]/components/recording-tab.tsx
  </files>
  <action>
The recording tab currently uses `readyRecording.blob_url` directly as the `<video>`/`<audio>` src (lines 110 and 119). This is a storage path like `recordings/calls/{id}/{file}.mp4`, not an accessible URL.

An endpoint exists: `GET /api/v2/calls/:id/recordings/:recordingId/playback-url` which returns `{ data: { url: string, expires_at: string } }` with a signed URL.

Changes needed:

1. Add state for the signed playback URL: `const [playbackUrl, setPlaybackUrl] = useState<string | null>(null)`.
2. Add a loading state: `const [loadingUrl, setLoadingUrl] = useState(false)`.
3. The component needs the `callId` — check if it's available. `readyRecording.call_id` exists in the `CallRecording` type. Use that.
4. Add `useAuth` from `@clerk/nextjs` and `createAuthenticatedClient` from `@/lib/api-client` imports.
5. Add a `useEffect` that fires when `readyRecording` is found:
   ```
   useEffect(() => {
     if (!readyRecording) return;
     setLoadingUrl(true);
     const fetchUrl = async () => {
       try {
         const token = await getToken();
         if (!token) return;
         const client = createAuthenticatedClient(token);
         const res = await client.get<{ data: { url: string } }>(
           `/calls/${readyRecording.call_id}/recordings/${readyRecording.id}/playback-url`
         );
         setPlaybackUrl(res.data.url);
       } catch (err) {
         console.error('Failed to load playback URL', err);
       } finally {
         setLoadingUrl(false);
       }
     };
     fetchUrl();
   }, [readyRecording?.id]);
   ```
6. Replace `readyRecording.blob_url!` on lines 110 and 119 with `playbackUrl!`.
7. Guard the player rendering: only show `<video>`/`<audio>` when `playbackUrl` is truthy. Show a loading indicator while `loadingUrl` is true.
8. Keep the existing time sync, speed controls, and duration display logic unchanged.

Do NOT change the `CallRecording` type or the `use-call-detail` hook — they already include `id` and `call_id`.
  </action>

Run `pnpm --filter @splits-network/portal build` — must compile. Grep recording-tab.tsx for `playback-url` to confirm the endpoint is called. Verify `blob_url` is no longer used as a media src.
  </verify>
