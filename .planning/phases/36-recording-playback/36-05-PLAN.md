<task type="auto">
  <name>Task 1: SAS URL endpoint for secure playback</name>
  <files>services/video-service/src/v2/interviews/recording-routes.ts</files>
  <action>
Add a new endpoint to the existing recording-routes.ts:

**GET /api/v2/interviews/:id/recording/playback-url** (authenticated)
- `requireUserContext(request)` to get clerkUserId
- Fetch interview by ID, validate recording_status is 'ready' and recording_blob_url exists
- Authorization check: caller must be an interview participant OR a company admin for the job's company
  - Fetch participants from repository, check if clerkUserId matches any participant's user_id
  - If not participant, check if user is company admin (query via application -> job -> company -> check user role)
  - If neither: return 403
- Generate a time-limited SAS URL for the Azure Blob:
  - Install `@azure/storage-blob` as dependency of video-service
  - Use `generateBlobSASQueryParameters` with:
    - `BlobSASPermissions.parse("r")` (read only)
    - `expiresOn: new Date(Date.now() + 60 * 60 * 1000)` (1 hour expiry)
    - `containerName` and `blobName` extracted from recording_blob_url
  - Construct the full SAS URL: `https://{accountName}.blob.core.windows.net/{containerName}/{blobPath}?{sasToken}`
- Return `{ data: { url: sasUrl, expires_at: expiresOn.toISOString() } }`

Also add:

**GET /api/v2/interviews/:id/recording/download-url** (authenticated)
- Same auth logic as playback-url
- Same SAS generation but with `content-disposition: attachment` header in the SAS
- Use `BlobSASPermissions.parse("r")` with `contentDisposition: "attachment; filename=interview-recording-{interviewId}.mp4"`
- Return `{ data: { url: downloadUrl, expires_at: expiresOn.toISOString() } }`

Both endpoints use the Azure config (accountName, accountKey, containerName) from the RecordingService or route config. Extract the SAS generation into a helper function `generateSasUrl(blobUrl, options)` to avoid duplication.

Add `@azure/storage-blob` to video-service package.json:
```bash
pnpm --filter @splits-network/video-service add @azure/storage-blob
```
  </action>
  <verify>Run `npx tsc --noEmit --project services/video-service/tsconfig.json` to verify types compile. Verify `@azure/storage-blob` is in package.json.</verify>
  <done>Playback and download URL endpoints return time-limited SAS URLs. Authorization checks ensure only participants and company admins can access recordings.</done>

**interview-recording-card.tsx** -- New component:
- Props: `{ interviewId: string; scheduledAt: string; recordingStartedAt: string; durationSeconds: number | null; participantNames: string[]; recordingStatus: string }`
- Shows a card with:
  - Border-left accent: `border-l-4 border-error` (recording = red accent)
  - Left side: `fa-duotone fa-regular fa-video text-error` icon
  - Title: "Interview Recording" (`text-sm font-black`)
  - Date: formatted scheduledAt (`text-sm text-base-content/60`)
  - Duration: formatted as "Xm Ys" (`text-sm text-base-content/60`)
  - Participants: comma-separated names (`text-sm text-base-content/50`)
  - Right side: Play button (`btn btn-primary btn-sm gap-2`) and Download button (`btn btn-ghost btn-sm`)
- **Expiry warning (Locked Decision #15):**
  - Calculate expiry date: `new Date(recordingStartedAt).getTime() + 90 * 24 * 60 * 60 * 1000`
  - Calculate days remaining: `Math.ceil((expiryDate - Date.now()) / (24 * 60 * 60 * 1000))`
  - If days remaining <= 7 and > 0: show a warning badge below the duration line:
    - `<span className="badge badge-warning badge-sm gap-1"><i className="fa-duotone fa-regular fa-triangle-exclamation" /> Expires in {daysRemaining} day(s)</span>`
  - If days remaining <= 0: show `<span className="badge badge-error badge-sm">Expired</span>` (recording may already be deleted by Azure lifecycle)
  - If days remaining > 7: no badge shown
- Play button: Opens `InterviewRecordingPlayer` in a DaisyUI modal (`dialog` element)
- Download button: Fetches download URL from `/api/v2/interviews/{id}/recording/download-url` and opens in new tab
- When `recordingStatus` is 'processing': Show "Processing..." with loading spinner instead of play/download buttons
- When `recordingStatus` is 'failed': Show "Recording failed" in error text
- Keep under 100 lines
