**pipeline-status.tsx**: Create a step-by-step pipeline status indicator.

Steps: Recording, Transcribing, Summarizing, Complete

Props: `{ recordings, transcript, summary }` (same types from use-call-detail)

Determine current step from artifact statuses:
- If no recordings with status 'ready' -> "Recording" step (or "No Recording" if no recordings at all)
- If recording ready but transcript is null or transcript_status is 'pending'/'processing' -> "Transcribing" step
- If transcript ready but summary is null or summary_status is 'pending'/'processing' -> "Summarizing" step
- If summary_status is 'ready' -> "Complete"
- If any status is 'failed' -> show failed state on that step

Render as a horizontal stepper using DaisyUI `steps` component:
```tsx
<ul className="steps steps-horizontal w-full">
  {STEPS.map((step, i) => (
    <li key={step.key} className={`step ${getStepClass(step, currentStep)}`}>
      {step.label}
    </li>
  ))}
</ul>
```
- Steps before current: `step-primary` (completed)
- Current step if processing: `step-primary` with a loading spinner alongside the label
- Steps after current: no modifier (upcoming)
- Failed step: `step-error`

Only render this component when the pipeline is in progress (not complete AND at least one recording exists). When complete, don't show the stepper — the summary tab content is sufficient.

**use-call-detail.ts**: Add auto-refresh behavior when pipeline is in progress.
- Detect pipeline in progress: recordings exist with status 'ready' but summary is null or summary_status is not 'ready'
- When in progress, set up a 15-second polling interval that calls `refetch`
- Clear interval when summary becomes ready or component unmounts
- Add `isPipelineProcessing` boolean to the hook return value

**call-detail-client.tsx**: Render PipelineStatus above the tabs when pipeline is processing:

```tsx
      transcript={call.transcript}
      summary={call.summary}
    />
  </div>
)}
```
Import the new `isPipelineProcessing` from useCallDetail and PipelineStatus component.
