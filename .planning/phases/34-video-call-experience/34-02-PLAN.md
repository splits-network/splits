  <action>
    Create four supporting components:

    **audio-level-meter.tsx** — Real-time mic level visualization for lobby (NOT in a LiveKit room):
    - Props: `{ audioTrack?: LocalAudioTrack }` from livekit-client
    - Use Web Audio API: create AudioContext, AnalyserNode from the track's mediaStream
    - Split frequency data into 5 bands, render as vertical bars
    - Bars use `bg-success` color, animate height based on volume level
    - Cleanup AudioContext on unmount
    - Show muted state when no audioTrack provided (gray bars, no animation)
    - See RESEARCH.md "Audio Level Meter (Bouncing Bars)" code example — follow that pattern

    **device-selector.tsx** — Device selection dropdowns:
    - Props: `{ kind: 'audioinput' | 'videoinput' | 'audiooutput', onDeviceChange: (deviceId: string) => void }`
    - Use `MediaDeviceSelect` from `@livekit/components-react` if it works outside LiveKitRoom context. If not (it likely requires room context), fall back to `navigator.mediaDevices.enumerateDevices()` manually.
    - If manual: `useEffect` to enumerate devices, filter by `kind`, render as DaisyUI `select` dropdown
    - Listen for `devicechange` event to update list when devices are plugged/unplugged
    - Label format: device label or "Device N" fallback if label is empty (happens before permission grant)

    **camera-off-fallback.tsx** — Avatar display when camera is off:
    - Props: `{ name: string, avatarUrl?: string }`
    - Dark background (`bg-base-300`), centered content
    - If avatarUrl: show DaisyUI avatar with `rounded-full`
    - If no avatarUrl: show initial letter in a `bg-primary text-primary-content rounded-full` circle
    - Name displayed below avatar in `text-lg font-medium`
    - See RESEARCH.md "Camera-Off Avatar State" code example

    **waiting-indicator.tsx** — Participant presence status:
    - Props: `{ participantName: string, isWaiting: boolean }`
    - When `isWaiting=true`: show "{name} is waiting" with a subtle pulse animation (CSS `animate-pulse` on a green dot)
    - When `isWaiting=false`: show "Waiting for {name}..." with a neutral dot
    - Clean, minimal design — this sits in the lobby info panel
  </action>

    - Use `usePreviewTracks` from `@livekit/components-react` for local camera/mic preview WITHOUT connecting to LiveKit server
    - CRITICAL: This component must NOT be inside a `<LiveKitRoom>`. The `usePreviewTracks` hook works independently.

    Layout (split screen, responsive):
    - Desktop (`lg:` and up): `grid grid-cols-1 lg:grid-cols-2 h-screen`
    - Left panel (`bg-base-300`): Camera preview video element (or CameraOffFallback when camera disabled)
      - Below video: AudioLevelMeter showing mic input level
      - Mute/unmute and camera on/off toggle buttons (DaisyUI `btn` with FontAwesome icons: `fa-microphone`/`fa-microphone-slash`, `fa-video`/`fa-video-slash`)
    - Right panel (`bg-base-100 p-6 lg:p-8`): Interview context info
      - Kicker: `text-sm font-semibold uppercase tracking-[0.2em] text-primary` showing interview type (e.g., "SCREENING INTERVIEW")
      - Job title in `text-2xl font-black`
      - Company name below job title
      - Scheduled time formatted nicely
      - Participant info: interviewer/candidate name + avatar
      - WaitingIndicator component
      - Device selectors (camera, mic, speaker) using DeviceSelector
      - "Join Interview" button: `btn btn-primary btn-lg w-full` at the bottom

    State management:
    - `audioEnabled` / `videoEnabled` booleans (toggle with buttons)

    - Handle `onError` from usePreviewTracks — show permission denied message with instructions

    Mobile: Left panel stacks on top (camera preview), right panel below.
