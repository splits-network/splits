  <action>
    Create a new shared package at `packages/shared-video/` following the same package.json/tsconfig.json patterns as `packages/shared-hooks/`.

    Package name: `@splits-network/shared-video`

    Install LiveKit dependencies:
    ```
    pnpm --filter @splits-network/shared-video add @livekit/components-react @livekit/components-styles livekit-client
    ```

    Also install in both consumer apps:
    ```
    pnpm --filter @splits-network/portal add @livekit/components-react @livekit/components-styles livekit-client
    pnpm --filter @splits-network/candidate add @livekit/components-react @livekit/components-styles livekit-client
    ```

    Add `@splits-network/shared-video` as a workspace dependency in both portal and candidate package.json files.

    Create these files:

    **src/types.ts** - Shared types:
    - `CallState = 'prep' | 'lobby' | 'connecting' | 'in-call' | 'post-call'`
    - `InterviewContext` — interview details returned by enriched token endpoint: `{ id, status, interview_type, title, scheduled_at, scheduled_duration_minutes, job: { id, title, company_name }, participants: Array<{ id, role, name, avatar_url }> }`
    - `TokenResult` — `{ jwt: string, room_name: string, interview: InterviewContext, participant: { id, role } }`
    - `LocalUserChoices` — `{ audioEnabled: boolean, videoEnabled: boolean, audioDeviceId?: string, videoDeviceId?: string }`

    **src/hooks/use-interview-token.ts** — Token fetching hook (see RESEARCH.md code example for pattern):
    - `useInterviewToken(apiBaseUrl: string)` returns `{ tokenData, error, loading, fetchAuthenticatedToken, exchangeMagicLink }`
    - `fetchAuthenticatedToken(interviewId: string)` — calls `POST /api/v2/interviews/:id/token` with credentials
    - `exchangeMagicLink(magicToken: string)` — calls `POST /api/v2/interviews/join` with token in body
    - Both return `TokenResult`

    **src/hooks/use-call-duration.ts** — Simple timer hook:
    - `useCallDuration()` returns `{ duration: number, formatted: string, start, stop, reset }`
    - Tracks elapsed seconds since `start()` called
    - `formatted` returns "MM:SS" string
    - Uses `useRef` for interval to avoid re-render issues

    **src/lib/livekit-config.ts** — LiveKit connection config:
    - Export `getLiveKitUrl()` that reads `NEXT_PUBLIC_LIVEKIT_URL` env var
    - Export default room options: `{ adaptiveStream: true, dynacast: true }`

    **src/index.ts** — Re-export all hooks, types, and config. Do NOT re-export React components yet (those come in later plans).
  </action>
