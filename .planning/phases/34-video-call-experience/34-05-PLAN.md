<task type="auto">
  <name>Task 2: Candidate interview page with magic link flow</name>
  <files>

  <action>
    Create the candidate interview route at `apps/candidate/src/app/(public)/interview/[token]/`.

    **layout.tsx** — Minimal layout for interview:
    - No navigation, full screen
    - Import `@livekit/components-styles` CSS
    - Just render `{children}`

    **page.tsx** — Server component:
    - Extract `token` from params (this is the magic link token, NOT the LiveKit JWT)
    - Render `<InterviewClient magicToken={token} />`
    - Metadata: `title: "Interview | Splits Network"`

    **interview-client.tsx** — Client component with state machine:
    - `'use client'` directive
    - Props: `{ magicToken: string }`
    - State: `callState: CallState`, starts at `'prep'` (candidate flow starts with prep page)
    - State: `livekitToken`, `userChoices`, `interviewContext`

    On mount:
    - Use `useInterviewToken` hook from shared-video
    - Call `exchangeMagicLink(magicToken)` to validate token and get interview context
    - Store `interviewContext` for prep page display
    - Handle errors: invalid/expired token shows error page ("This interview link is no longer valid")

    State machine (same pattern as portal but starts at 'prep'):
    ```
    'prep' -> <PrepPage interviewContext={interviewContext} onReady={() => setCallState('lobby')} />
    'lobby' -> <VideoLobby interviewContext={interviewContext} onJoin={handleJoin} />
    'connecting' | 'in-call' -> <LiveKitRoom ...><VideoRoom ... /></LiveKitRoom>
    'post-call' -> <PostCallSummary duration={duration} isCandidate={true} onClose={handleClose} />
    ```

    `handleJoin`:
    - Uses the LiveKit JWT already received from `exchangeMagicLink` (stored in tokenData.jwt)
    - Sets `livekitToken`, `userChoices`, transitions to `'connecting'`
    - NOTE: The magic link exchange returns both the LiveKit JWT and interview context in one call

    `handleClose`:
    - For candidates: just show "Thank you" — no auto-redirect, no window.close()
    - Candidate may want to close the tab themselves

    Error states:
    - Invalid/expired magic link token: "This interview link is no longer valid. Please contact your recruiter for a new link."
    - Interview cancelled: "This interview has been cancelled."
    - Network error: retry button

    CRITICAL: The magic link token in the URL is NOT the LiveKit JWT. The flow is:
    1. Candidate opens `/interview/{magic-token}`
    2. `exchangeMagicLink(magic-token)` calls `POST /api/v2/interviews/join` with the magic token
    3. Server validates magic token, returns LiveKit JWT + interview context
    4. LiveKit JWT is used later with `<LiveKitRoom token={livekitJwt}>`
    See RESEARCH.md Pitfall 7.
