**Create apps/portal/src/app/portal/applications/[id]/components/interview-transcript-panel.tsx:**

Client component (`'use client'`) that displays the transcript below the recording player.

Props: `{ interviewId: string, segments: TranscriptSegment[], onTimestampClick?: (seconds: number) => void }`

Import `TranscriptSegment` from `@splits-network/shared-types`.

Rendering:
1. Collapsible section using DaisyUI `collapse collapse-arrow`:
   - Title: "Transcript" with segment count badge
   - Content: scrollable list of segments

2. Each segment renders as:
   ```tsx
   <div className="flex gap-3 py-2 border-b border-base-200 last:border-0">
     <button
       onClick={() => onTimestampClick?.(segment.start)}
       className="text-sm font-mono text-primary hover:underline shrink-0 w-16"
     >
       {formatTimestamp(segment.start)}
     </button>
     <span className="text-sm font-semibold text-secondary shrink-0 w-24 truncate">
       {segment.speaker}
     </span>
     <p className="text-sm flex-1">{segment.text}</p>
   </div>
   ```

3. Helper `formatTimestamp(seconds: number): string` — format as `MM:SS` (or `H:MM:SS` if >= 3600).

4. The `onTimestampClick` callback is designed to integrate with the recording player (Phase 38 can wire this to seek the video). For now, the recording card can pass a no-op or undefined.

5. If segments is empty, show "No transcript available" in muted text.

Keep under 80 lines.
