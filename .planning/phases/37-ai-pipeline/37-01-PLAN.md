Create migration with three parts:

**Part 1: interview_transcripts table**
```sql
CREATE TABLE interview_transcripts (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    interview_id UUID NOT NULL REFERENCES interviews(id) ON DELETE CASCADE,
    full_text TEXT NOT NULL,
    segments JSONB NOT NULL DEFAULT '[]',
    language TEXT DEFAULT 'en',
    whisper_model TEXT,
    processing_time_ms INT,
    created_at TIMESTAMPTZ NOT NULL DEFAULT now()
);
CREATE UNIQUE INDEX idx_interview_transcripts_interview ON interview_transcripts(interview_id);
```

The `segments` JSONB stores an array of objects: `{ start: number, end: number, text: string, speaker: string }` where start/end are seconds from recording start, and speaker is the participant name or role.

**Part 2: Add interview_summary to application_notes note_type CHECK constraint**
Follow the exact pattern from `20260223003550_add_pitch_note_type.sql`:
- DROP existing constraint
- ADD new constraint with all existing values plus `'interview_summary'`
- Update COMMENT on column

**Part 3: Add AI pipeline status columns to interviews table**
```sql
ALTER TABLE interviews
    ADD COLUMN transcript_status TEXT DEFAULT NULL
        CHECK (transcript_status IN ('pending', 'transcribing', 'summarizing', 'posting', 'complete', 'failed')),
    ADD COLUMN transcript_error TEXT;
```

IMPORTANT: The CHECK constraint MUST include 'posting' — this is a locked decision from CONTEXT.md requiring granular step display: "Transcribing audio..." -> "Generating summary..." -> "Posting to notes...". The 'posting' status represents the final step before 'complete'.
