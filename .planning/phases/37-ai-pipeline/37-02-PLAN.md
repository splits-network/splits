Methods:
1. `getInterviewWithContext(interviewId: string)` — Query interviews table joined with interview_participants (with user names), and the application's job (title, description, recruiter_description). Returns `{ interview, participants: { name, role }[], job: { title, description }, application_id }`. Uses sequential queries (consistent with video-service pattern from Phase 34-01 decision).
2. `updatePipelineStatus(interviewId: string, status: string, error?: string)` — Update `transcript_status` and optionally `transcript_error` on interviews table.
3. `saveTranscript(data: { interview_id, full_text, segments, language, whisper_model, processing_time_ms })` — Insert into interview_transcripts table. Return the created row.
4. `postSummaryNote(data: { application_id, message_text })` — Insert into application_notes with: `note_type: 'interview_summary'`, `created_by_type: 'platform_admin'`, `visibility: 'shared'`, `created_by_user_id` set to a system user ID queried from users table where email matches a system identifier (or the interview creator). Use the interview's `created_by` user_id as the note author since we need a valid FK. This makes the note appear as if posted by the interviewer but with the AI badge distinguishing it.
5. `getRecordingSignedUrl(interviewId: string)` — Query interviews for `recording_blob_url`, then use Supabase Storage client to generate a signed download URL (same pattern as video-service's signed-url-helper.ts). Bucket name: `'interview-recordings'`. Extract storage path from blob URL (handle both relative paths like `recordings/id/file.mp4` and full URLs).

Constructor takes `SupabaseClient` and `Logger`.

**Create services/ai-service/src/v2/transcription/summarizer.ts:**

SummaryService class that generates structured summaries from transcripts.

Constructor takes `Logger`. Reads `OPENAI_API_KEY` and model from env (default `gpt-4o-mini`).

Method `generateSummary(transcript: string, jobTitle: string, jobDescription: string, participants: { name: string, role: string }[]): Promise<string>`:

1. Build prompt that instructs GPT to produce a recruiter-focused debrief summary with these sections:
   - **Key Discussion Points** — 3-5 bullet points of main topics covered
   - **Strengths** — 3-5 bullet points mapping candidate responses to job requirements
   - **Concerns** — 0-3 bullet points of gaps or weak areas
   - **Overall Impression** — One soft sentence like "Overall positive impression" or "Mixed results with strong technical foundation" — NOT a hiring recommendation

2. Prompt instructions:
   - "You are summarizing a video interview transcript for a recruiter"
   - "The candidate interviewed for the role: {jobTitle}"
   - "Job description: {jobDescription}" (truncate to 2000 chars)
   - "Participants: {list with roles}"
   - "Keep total summary to 200-300 words"
   - "Use bullet points and short phrases, not paragraphs"
   - "Map candidate responses to job requirements where possible"
   - "Do NOT make explicit hire/no-hire recommendations"
   - Response format: plain text with markdown headers (##), not JSON

3. Call OpenAI chat completions API with temperature 0.3, max_tokens 1500. Use the same retry pattern as AIReviewServiceV2 (2 retries with exponential backoff for 429/5xx).

4. Return the summary text string directly (it will be stored as message_text in the note).

**Create services/ai-service/src/v2/transcription/service.ts:**

TranscriptionPipelineService class — orchestrates the full pipeline.

Constructor takes `TranscriptionRepository`, `SummaryService`, `IEventPublisher | undefined`, `Logger`.

Method `processRecording(event: { interview_id: string, recording_url: string, duration_seconds: number, file_size_bytes: number })`:

1. **Set status to 'pending':** `repository.updatePipelineStatus(interview_id, 'pending')`

2. **Get interview context:** `repository.getInterviewWithContext(interview_id)` — need application_id, job info, participant names

3. **Set status to 'transcribing':** `repository.updatePipelineStatus(interview_id, 'transcribing')`

4. **Download recording:** Get signed URL via `repository.getRecordingSignedUrl(interview_id)`. Fetch the recording as an ArrayBuffer.

5. **Handle >25MB Whisper limit:** Check `file_size_bytes`. If >25MB (25 * 1024 * 1024), the recording exceeds Whisper's limit. In this case:
   - Use FFmpeg to extract audio only: spawn `ffmpeg -i pipe:0 -vn -acodec libopus -b:a 48k -f ogg pipe:1` piping the video buffer in and getting audio buffer out
   - This typically reduces a 720p 30min recording from ~100MB to ~5-10MB
   - If FFmpeg is not available or fails, log error and attempt Whisper with original file anyway (it may reject it)
   - Add `fluent-ffmpeg` is NOT needed — use Node child_process.spawn with pipes

   If <=25MB, use the downloaded buffer directly.

6. **Call Whisper API:** Send audio to `POST https://api.openai.com/v1/audio/transcriptions` as multipart form data:
   - `model`: `whisper-1`
   - `file`: the audio buffer as a File/Blob with name `recording.mp4` (or `recording.ogg` if extracted)
   - `response_format`: `verbose_json` (gives word-level timestamps)
   - `timestamp_granularities[]`: `segment`
   - Use fetch with FormData (Node 18+ native FormData)
   - Timeout: 300 seconds (5 min for long recordings)
   - Retry: 2 retries with exponential backoff for 429/5xx (same pattern as review service)

7. **Map Whisper segments to speaker labels:** Whisper returns segments with `{ start, end, text }`. Map speaker labels by interleaving: use a simple heuristic — odd segments = interviewer, even segments = candidate (or use the participant names). Store as `TranscriptSegment[]`.

   Better approach: Since this is a 1:1 interview, use the participant list from the interview context. Label all segments generically for now; speaker diarization can be enhanced later. Set speaker to the participant names alternating or just "Speaker" with timestamps.

8. **Save transcript:** `repository.saveTranscript({ interview_id, full_text: whisperResponse.text, segments, language: whisperResponse.language, whisper_model: 'whisper-1', processing_time_ms })`

9. **Set status to 'summarizing':** `repository.updatePipelineStatus(interview_id, 'summarizing')`

10. **Generate summary:** `summaryService.generateSummary(whisperResponse.text, job.title, job.description, participants)`

11. **Set status to 'posting':** `repository.updatePipelineStatus(interview_id, 'posting')`
    This is a locked decision from CONTEXT.md — the granular step display requires "Posting to notes..." as a visible pipeline step.

12. **Post summary note:** `repository.postSummaryNote({ application_id: context.application_id, message_text: summary })`

13. **Set status to 'complete':** `repository.updatePipelineStatus(interview_id, 'complete')`

14. **Publish event (non-critical):** If eventPublisher exists, publish `interview.transcript_ready` with `{ interview_id, application_id, transcript_id }`. Wrap in try/catch.

**Error handling:** Wrap entire pipeline in try/catch. On failure:
- `repository.updatePipelineStatus(interview_id, 'failed', error.message)`
- Log error with full context
- Do NOT rethrow (don't nack the RabbitMQ message — failed transcription should not retry indefinitely)

Keep file under 200 lines.

```typescript
// GET /api/v2/interviews/:id/transcript - Get transcript with segments
app.get('/api/v2/interviews/:id/transcript', async (request, reply) => {
    try {
        requireUserContext(request);
        const { id } = request.params as { id: string };

        const transcript = await repository.getTranscriptByInterviewId(id);
        if (!transcript) {
            return reply.code(404).send({ error: 'No transcript found for this interview' });
        }

        return reply.send({ data: transcript });
    } catch (error: any) {
        return reply
            .code(error.statusCode || 400)
            .send({ error: error.message });
    }
});
```

**Add `getTranscriptByInterviewId` method to services/video-service/src/v2/interviews/repository.ts:**

```typescript
async getTranscriptByInterviewId(interviewId: string): Promise<any | null> {
    const { data, error } = await this.supabase
        .from('interview_transcripts')
        .select('id, interview_id, full_text, segments, language, whisper_model, processing_time_ms, created_at')
        .eq('interview_id', interviewId)
        .single();

    if (error && error.code === 'PGRST116') return null; // No rows
    if (error) throw error;
    return data;
}
```
