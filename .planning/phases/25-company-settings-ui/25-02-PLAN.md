  <name>Task 1: Create CompanyTagsSection component</name>
  <files>apps/portal/src/components/basel/company-settings/company-tags-section.tsx</files>
  <action>
Create a new "use client" component that renders three BaselSkillPicker instances for tech stack skills, perks, and culture tags. Follow the exact pattern from `apps/candidate/src/app/portal/profile/components/section-skills.tsx`.

**Props interface:**
```typescript
interface CompanyTagsSectionProps {
    companyId: string;
}
```

**Component structure:**

1. **Imports:**
   - `useState, useCallback, useEffect` from react
   - `useAuth` from `@clerk/nextjs`
   - `createAuthenticatedClient` from `@/lib/api-client`
   - `BaselSkillPicker, type SkillOption` from `@splits-network/basel-ui`

2. **State:** Three separate `SkillOption[]` states: `skills`, `perks`, `cultureTags`. Three `loading` booleans (or a single object). One `saving` string to track which picker is saving (e.g., `"skills" | "perks" | "culture-tags" | null`).

3. **Load existing data on mount** (single useEffect, three parallel fetches):
   ```typescript
   useEffect(() => {
       async function load() {
           const token = await getToken();
           if (!token) return;
           const client = createAuthenticatedClient(token);

           const [skillsRes, perksRes, cultureRes] = await Promise.all([
               client.get(`/company-skills?company_id=${companyId}`),
               client.get(`/company-perks?company_id=${companyId}`),
               client.get(`/company-culture-tags?company_id=${companyId}`),
           ]);

           // Extract the joined entity from each junction row
           setSkills((skillsRes.data || []).filter((r: any) => r.skill).map((r: any) => r.skill));
           setPerks((perksRes.data || []).filter((r: any) => r.perk).map((r: any) => r.perk));
           setCultureTags((cultureRes.data || []).filter((r: any) => r.culture_tag).map((r: any) => r.culture_tag));
       }
       load().finally(() => setLoading(false));
   }, [companyId]); // eslint-disable-line react-hooks/exhaustive-deps
   ```

4. **Search functions** (three useCallbacks, same pattern as section-skills.tsx):
   - `searchSkills`: `GET /skills?q=${query}&limit=10` — returns `response.data`
   - `searchPerks`: `GET /perks?q=${query}&limit=10` — returns `response.data`
   - `searchCultureTags`: `GET /culture-tags?q=${query}&limit=10` — returns `response.data`

5. **Create functions** (three useCallbacks):
   - `createSkill`: `POST /skills` with `{ name }` — returns `response.data`
   - `createPerk`: `POST /perks` with `{ name }` — returns `response.data`
   - `createCultureTag`: `POST /culture-tags` with `{ name }` — returns `response.data`

6. **Change handlers with auto-save** (three useCallbacks, follow candidate section-skills.tsx pattern):
   Each handler: (a) updates local state immediately, (b) calls bulk-replace PUT:
   - `handleSkillsChange`: `PUT /companies/${companyId}/skills` with body `{ skills: newSkills.map(s => ({ skill_id: s.id })) }`
   - `handlePerksChange`: `PUT /companies/${companyId}/perks` with body `{ perks: newPerks.map(p => ({ perk_id: p.id })) }`
   - `handleCultureTagsChange`: `PUT /companies/${companyId}/culture-tags` with body `{ culture_tags: newTags.map(t => ({ culture_tag_id: t.id })) }`

7. **Render three pickers** in a `space-y-8` container, each with:
   - A heading (`h3 text-lg font-bold`) and description (`text-sm text-base-content/50`)
   - BaselSkillPicker with appropriate props
   - Saving indicator (small "Saving..." text like section-skills.tsx)

   Sections:
   - **Tech Stack** — label "Tech Stack", placeholder "Search for a technology...", maxSkills=30
   - **Perks & Benefits** — label "Perks & Benefits", placeholder "Search for a perk...", maxSkills=30
   - **Culture Tags** — label "Culture & Values", placeholder "Search for a culture tag...", maxSkills=20

8. **Loading state:** Show spinner while loading initial data (same pattern as section-skills.tsx).

**Important:** The `SkillOption` interface from BaselSkillPicker expects `{ id, name, slug }`. Perks and CultureTags have the same shape (id, name, slug), so they are compatible with SkillOption without mapping. Skills also have this shape. No type adapters needed.
  </action>
