<objective>
Add gateway auth bypasses for 4 candidate/webhook routes and configure LiveKit webhook URL.

Purpose: Candidate magic-link flows (reschedule, available-slots, recording consent) and the LiveKit Egress webhook are blocked by the API gateway's Clerk auth check. These routes handle their own authentication (magic link token validation or LiveKit signature verification) and must skip the gateway auth layer. Additionally, LiveKit Egress has no configured URL to POST completion events, breaking the entire recording-to-AI pipeline.

Output: Gateway allows unauthenticated requests through for the 4 specified routes; LiveKit config includes webhook URL.
</objective>
