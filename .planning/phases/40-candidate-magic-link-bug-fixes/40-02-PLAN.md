  <action>
In the GET `/api/v2/interviews/:id/notes` handler (lines 531-558), add support for a `mine` query parameter that filters to just the authenticated user's note.

Current behavior: Returns all notes as `{ data: InterviewNoteWithUser[] }`.
New behavior: When `?mine=true` is in the query, return only the current user's note as `{ data: InterviewNoteWithUser | null }`.

Implementation:
1. Parse `mine` from query params: `const query = request.query as { token?: string; mine?: string };`
2. After fetching all notes, if `query.mine === 'true'`, filter to the current user's note:
   - For magic link path: resolve participantId from tokenResult, filter `notes.find(n => n.participant_id === tokenResult.participant.id)`
   - For Clerk path: resolve userId via `requireUserContext` + `repository.resolveUserId`, filter `notes.find(n => n.user_id === userId)`
3. Return `{ data: matchedNote || null }` when mine=true, `{ data: notes }` otherwise.

The handler already resolves the user identity for both auth paths (magic link token validation at line 537, Clerk requireUserContext at line 548). Extend these blocks to capture participant/user ID, then filter after fetching notes.

Concrete structure:
```typescript
app.get('/api/v2/interviews/:id/notes', async (request, reply) => {
    try {
        const { id } = request.params as { id: string };
        const query = request.query as { token?: string; mine?: string };

        let userId: string | null = null;
        let participantId: string | null = null;

        if (query?.token) {
            const tokenResult = await tokenService.validateMagicLinkToken(query.token);
            if (!tokenResult) {
                return reply.code(401).send({ error: 'Invalid or expired token' });
            }
            if (tokenResult.interview.id !== id) {
                return reply.code(403).send({ error: 'Token does not match this interview' });
            }
            participantId = tokenResult.participant.id;
            userId = tokenResult.participant.user_id;
        } else {
            const { clerkUserId } = requireUserContext(request);
            userId = await repository.resolveUserId(clerkUserId);
        }

        const notes = await repository.getInterviewNotes(id);

        if (query?.mine === 'true') {
            const myNote = notes.find(n =>
                participantId ? n.participant_id === participantId : n.user_id === userId
            );
            return reply.send({ data: myNote || null });
        }

        return reply.send({ data: notes });
    } catch (error: any) {
        return reply.code(error.statusCode || 400).send({ error: error.message });
    }
});
```

IMPORTANT: Do NOT break the existing array response for calls without `mine=true`. The `post-to-application` flow uses `getInterviewNotes` directly via the repository, not this endpoint, so it is unaffected.
