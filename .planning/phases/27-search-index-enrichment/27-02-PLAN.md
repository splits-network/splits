  <action>
Create a migration with three trigger functions and their corresponding triggers. The strategy is simple: when a junction record changes, "touch" the parent company by updating its `updated_at` timestamp. This fires the existing `update_companies_search_vector` BEFORE trigger (which now aggregates junction data from Plan 27-01) and the `sync_company_to_search_index` AFTER trigger, keeping everything in sync.

**1. Company Skills cascade:**
```sql
CREATE OR REPLACE FUNCTION search.cascade_company_skills_to_search() RETURNS trigger AS $$
DECLARE
    v_company_id UUID;
BEGIN
    -- Get company_id from either NEW (insert) or OLD (delete)
    v_company_id := COALESCE(NEW.company_id, OLD.company_id);

    -- Touch the company to re-fire its search triggers
    UPDATE public.companies
    SET updated_at = now()
    WHERE id = v_company_id;

    RETURN COALESCE(NEW, OLD);
END;
$$ LANGUAGE plpgsql;

CREATE TRIGGER cascade_company_skills_to_search_insert
AFTER INSERT ON public.company_skills
FOR EACH ROW
EXECUTE FUNCTION search.cascade_company_skills_to_search();

CREATE TRIGGER cascade_company_skills_to_search_delete
AFTER DELETE ON public.company_skills
FOR EACH ROW
EXECUTE FUNCTION search.cascade_company_skills_to_search();
```

**2. Company Perks cascade:**
Same pattern — create `search.cascade_company_perks_to_search()` and two triggers (INSERT + DELETE) on `company_perks`.

**3. Company Culture Tags cascade:**
Same pattern — create `search.cascade_company_culture_tags_to_search()` and two triggers (INSERT + DELETE) on `company_culture_tags`.

**Design notes:**
- Use separate functions (not one shared function) because trigger functions on different tables may need different column references in the future. This follows the established pattern in `20260213000004` where each entity type has its own sync function.
- No UPDATE trigger needed on junction tables — they have composite PKs and no mutable columns besides the FK pair.
- The "touch" strategy is elegant because it reuses the enriched BEFORE/AFTER triggers from Plan 27-01 without duplicating search vector computation logic in the cascade functions.
