# v12.0 Milestone Audit: Per-Call Recording & AI Controls

## Milestone Definition of Done

Recording and AI analysis are per-call opt-in toggles gated by the creator's subscription tier, replacing call_type-driven behavior with explicit user choice and tiered access.

## Success Criteria Evaluation

| # | Criterion | Status | Evidence |
|---|-----------|--------|----------|
| 1 | Call creation form has "Record this call" toggle (off by default) and "AI Analysis" toggle for Partner only | **Pass** | `recording-controls.tsx` renders tier-branched toggles; `recording_enabled` defaults false |
| 2 | Free sees combined upgrade prompt; Pro sees AI-only upgrade; Partner sees normal toggles | **Pass** | `RecordingControls` branches on `planTier` with three paths: starter/pro/partner |
| 3 | AI pipeline checks creator's subscription tier before processing | **Pass** | `ai-service/call-pipeline/service.ts` calls `getCreatorTier()` and gates each step |
| 4 | Free tier recordings auto-delete after 7 days with 2-day email warning | **Pass** | `expire-recording-cleanup.ts` sends warning at 5d, deletes at 7d; K8s CronJob at 3 AM UTC |
| 5 | Call type selector visible with context-inferred defaults | **Pass** | `call-type-selector.tsx` with `inferCallType()` and 4 call types |
| 6 | Post-call detail shows upgrade prompts on locked tabs | **Pass** | `LockedTabUpgrade` component; transcript locked for starter, summary locked for starter+pro |

### Type Gap (non-critical)

`apps/portal/src/hooks/use-create-call.ts` — `CreateCallPayload` interface does not declare `recording_enabled`, `transcription_enabled`, or `ai_analysis_enabled`. The runtime payload from `call-creation-modal.tsx` includes them (JS object spread), so the single consumer works. But the type contract is incomplete — any future callers building payloads from the interface would silently omit the flags.

## Tech Debt

### Phase 53

1. **Stale CALL_TYPE_LABELS map** — `apps/portal/src/app/portal/calls/types.ts` has old slugs (`recruiter_company`, `screen`, `debrief`, `check_in`, `general`). New slugs from `call-type-selector.tsx` (`interview`, `recruiting_call`, `client_meeting`, `internal_call`) fall through to the generic `formatCallType()` string formatter. Works but is dead code.

2. **`any` type in cleanup job** — `expire-recording-cleanup.ts` uses `any` for the supabase parameter due to complex nested join type inference. Runtime correct.

3. **Implicit PostgREST FK join** — The cleanup job's `subscriptions:created_by` join pattern depends on PostgREST resolving `calls.created_by → subscriptions.user_id` implicitly. Should verify against production schema to ensure tier detection doesn't silently fail.

4. **Missing VERIFICATION.md files** — None of the 4 plans have formal verification documents. Inline verification in summaries confirms builds pass.

### Total: 4 items across 1 phase
