V2 to V3 Migration Plan

Chat Reports Resource

Service: chat-service | Table: chat_reports, chat_moderation_audit | Generated: 2026-03-17

1. V2 API Surface

The "reports" resource spans two concerns: user-facing report submission and admin moderation (reports queue, evidence, actions, audit log, metrics). All currently live in the monolithic v2/chat/routes.ts file.

MethodV2 PathPurposeAuthTables
POST /api/v2/chat/reports Submit a report (user-facing) Required chat_reports, chat_messages (evidence)
GET /api/v2/admin/chat/reports List all reports (admin only) Required + isPlatformAdmin chat_reports
GET /api/v2/admin/chat/reports/:id/evidence Get report with evidence messages Required + isPlatformAdmin chat_reports, chat_messages
POST /api/v2/admin/chat/reports/:id/action Take moderation action on report Required + isPlatformAdmin chat_reports, chat_moderation_audit
GET /api/v2/admin/chat/audit List moderation audit log Required + isPlatformAdmin chat_moderation_audit
GET /api/v2/admin/chat/metrics Admin dashboard metrics Required + isPlatformAdmin Multiple tables (counts)
Note: V3 aliases already exist for POST /api/v3/chat/reports in the V2 routes file (line 902). This alias calls the same V2 service. The V3 gateway has a proxy entry for it. The admin endpoints have no V3 aliases yet.

Path prefix: The shared API client defaults to /api/v3 prefix. Frontend calls like client.get('/admin/chat/reports') become GET /api/v3/admin/chat/reports. Currently these have NO gateway route in V3 -- they likely fall through to the V2 catch-all (/api/v2/chat/*) via some path rewriting, or are broken. The V3 migration must add proper gateway routes.

3. Migration Classification

V2 RouteV3 CategoryV3 PathReason
POST /v2/chat/reports Action POST /v3/chat/reports/actions/submit Side-effect: creates report + collects evidence messages. Not CRUD (no list/get/update/delete for normal users).
GET /v2/admin/chat/reports Core CRUD GET /v3/chat/reports Standard list of reports (flat, single table). Admin-only via service layer.
GET /v2/admin/chat/reports/:id Core CRUD GET /v3/chat/reports/:id Standard get by ID (flat). Does not exist in V2 but needed for Core 5.
GET /v2/admin/chat/reports/:id/evidence View GET /v3/chat/reports/:id/view/evidence Joins chat_reports + chat_messages. Shaped response with evidence bundle.
POST /v2/admin/chat/reports/:id/action Action POST /v3/chat/reports/:id/actions/moderate Side-effect: updates report status + creates moderation audit record.
N/A (Core 5 requirement) Core CRUD POST /v3/chat/reports Core 5 POST. Redirects to /actions/submit (same pattern as conversations).
N/A (Core 5 requirement) Core CRUD PATCH /v3/chat/reports/:id Core 5 PATCH. Admin updates report status directly (simple field update).
N/A (Core 5 requirement) Core CRUD DELETE /v3/chat/reports/:id Core 5 DELETE. Soft-delete a report (admin only).
Scope note: The admin audit log (GET /admin/chat/audit) and admin metrics (GET /admin/chat/metrics) are separate resources, not part of the reports migration. They should be migrated as their own resources in a future pass.
