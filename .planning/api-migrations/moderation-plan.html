V2 to V3 Migration Plan: Chat Moderation Resource

Service: chat-service  |  Date: 2026-03-17  |  Status: Plan Ready

Context: "Moderation" is not a single DB table -- it is a domain grouping of admin-only operations spanning chat_reports, chat_moderation_audit, and admin metrics aggregation. In V2 these live as routes under /api/v2/admin/chat/* inside the monolithic chat routes file. The V3 migration extracts them into a standalone moderation resource with proper CRUD, views, and actions.

Important: The portal frontend uses createAuthenticatedClient which has a default path prefix of /api/v3. The admin chat moderation page calls /admin/chat/reports which resolves to /api/v3/admin/chat/reports. Currently these routes are NOT in the V3 gateway but ARE handled by v2 chat route aliases in the v2 routes file. The V3 migration will create proper standalone routes and gateway entries under the /api/v3/moderation/* namespace.

3. Migration Classification

V2 RouteV3 CategoryV3 PathReason
GET /v2/admin/chat/reports Core CRUD GET /v3/moderation/reports Standard list of chat_reports, flat data, admin-only
GET /v2/admin/chat/reports/:id/evidence View GET /v3/moderation/reports/:id/view/evidence Joins report + messages -- shaped data for evidence review
POST /v2/admin/chat/reports/:id/action Action POST /v3/moderation/reports/:id/actions/take-action Side effects: creates audit entry, updates report status
GET /v2/admin/chat/audit Core CRUD GET /v3/moderation/audit-log Standard list of chat_moderation_audit, flat data, admin-only
GET /v2/admin/chat/metrics View GET /v3/moderation/views/metrics Aggregation across multiple tables -- shaped admin dashboard data
Design decision: "Moderation" becomes the top-level resource namespace at /api/v3/moderation/. Under it, reports and audit-log are sub-resources with their own Core 5 CRUD. The metrics view and evidence view are views under those sub-resources. This is cleaner than /admin/chat/* because the admin check happens in the service layer via isPlatformAdmin, not via URL path.

Risk Assessment

  • Low risk: All admin-only endpoints. No public or cross-service consumers.
  • Low risk: User-facing report/block already V3-proxied -- not part of this migration.
  • Medium risk: The frontend currently calls /api/v3/admin/chat/* via v2 alias routes. Need to ensure the new V3 routes work before updating frontend URLs.
