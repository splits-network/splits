## Critical Integration Gaps

### Gap 1: Missing Gateway Auth Bypasses (4 routes)

**Impact:** Candidate-facing features that use magic link tokens are blocked by the API gateway's Clerk authentication check.

**Routes needing auth bypass in `services/api-gateway/src/index.ts`:**
1. `POST /api/v2/interviews/*/reschedule-request` — Candidate reschedule via magic link
2. `GET /api/v2/interviews/*/available-slots` — Candidate slot fetch with token query param
3. `POST /api/v2/interviews/*/recording/consent` — Candidate recording consent with magic link token
4. `POST /api/v2/interviews/recording/webhook` — LiveKit Egress callback (no browser auth)

**Breaks:** Candidate reschedule flow, candidate recording consent, recording completion pipeline

### Gap 2: LiveKit Webhook URL Not Configured

**Impact:** LiveKit Egress has no configured URL to POST completion events. When recording finishes, LiveKit cannot notify video-service.

**Fix:** Add webhook section to `infra/k8s/livekit/livekit-config.yaml`:
```yaml
webhook:
  urls:
    - http://video-service:3019/api/v2/interviews/recording/webhook
```

**Breaks:** Entire Phase 36 → Phase 37 data flow. Recording completion → AI transcription → summary posting is completely disconnected.

### Gap 3: Candidate Notes Endpoints Require Clerk Auth

**Impact:** The interview notes endpoints (PUT/GET/POST) call `requireUserContext(request)` which requires `x-clerk-user-id` header. Candidates using magic links don't have Clerk auth.

**Fix:** Add dual-auth support (magic link token OR Clerk) to notes routes, following the pattern used in recording consent endpoint.

**Breaks:** Candidate in-call note-taking and auto-post on disconnect.
