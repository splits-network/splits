## Bugs Found

### BUG-01: Candidate Recording Consent Token Location (Medium)

**Location:** `packages/shared-video/src/hooks/use-recording-state.ts` lines 55-71
**Problem:** `submitConsent(magicToken)` sends token as `Authorization: Bearer` header, but backend at `services/video-service/src/v2/interviews/recording-routes.ts` line 60 reads `body?.token`. Token isn't in body, falls through to Clerk auth which candidates don't have.
**Impact:** Candidates cannot submit recording consent via magic link.
**Fix:** Send magic token in request body as `{ token }` when provided, not as Bearer header.

### BUG-02: Notes Load-on-Mount Array Mismatch (Low)

**Location:** `packages/shared-video/src/hooks/use-interview-notes.ts` line 83
**Problem:** `GET /interviews/:id/notes` returns `{ data: InterviewNoteWithUser[] }` (array), but hook reads `result.data?.content` as if single object. Notes never restore on page load.
**Impact:** Notes don't persist across refresh/reconnect. Save still works correctly.
**Fix:** Read array response and find current user's note from the list.
